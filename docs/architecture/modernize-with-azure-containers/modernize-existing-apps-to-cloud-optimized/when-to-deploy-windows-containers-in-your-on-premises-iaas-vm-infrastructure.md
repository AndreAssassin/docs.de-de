# <a name="when-to-deploy-windows-containers-in-your-on-premises-iaas-vm-infrastructure"></a>Umstände, unter denen eine Bereitstellung von Windows-Containern in Ihrer lokalen IaaS-VM-Infrastruktur erfolgen sollte

- Ihre Organisation ist möglicherweise nicht bereit für den Umstieg in die Cloud, oder Sie ist möglicherweise nicht in der Lage, aus geschäftlichen Gründen in die Cloud zu wechseln. Allerdings können Sie weiterhin die Vorteile der Verwendung von Windows-Containern in ihren eigenen Rechenzentren nutzen.

- Möglicherweise verfügen Sie über andere Artefakte, die lokal verwendet werden, und die Sie möglicherweise verlangsamen, wenn Sie versuchen, in die Cloud zu wechseln. Beispielsweise Sicherheits-oder Authentifizierungs Abhängigkeiten mit lokalen Windows Server-Active Directory oder anderen lokalen Medienobjekten.

- Wenn Sie noch heute Windows-Container verwenden, können Sie von einer viel besseren Position aus eine phasenweise Migration in die Cloud durchführen. Windows-Container werden zu einer Bereitstellungs Einheit für jede Cloud, ohne dass eine Sperre erfolgt.
