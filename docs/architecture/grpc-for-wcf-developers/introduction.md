## <a name="history"></a><span data-ttu-id="1289c-112">Versionsgeschichte</span><span class="sxs-lookup"><span data-stu-id="1289c-112">History</span></span>

<span data-ttu-id="1289c-113">Das Grundprinzip eines Computernetzwerks ist nicht mehr als eine Gruppe von Computern, die Daten untereinander austauschen, um eine Reihe von zusammenhängenden Aufgaben zu erreichen, die sich seit der Inbetriebnahme nicht geändert haben.</span><span class="sxs-lookup"><span data-stu-id="1289c-113">The fundamental principle of a computer network as nothing more than a group of computers exchanging data with each other to achieve a set of interrelated tasks hasn't changed since its inception.</span></span> <span data-ttu-id="1289c-114">Komplexität, Skalierung und Erwartungen wurden jedoch exponentiell vergrößert.</span><span class="sxs-lookup"><span data-stu-id="1289c-114">However, the complexity, scale, and expectations have grown exponentially.</span></span>  

<span data-ttu-id="1289c-115">In den 90er Jahren lag der Schwerpunkt hauptsächlich auf der Verbesserung interner Netzwerke mit derselben Sprache und denselben Plattformen.</span><span class="sxs-lookup"><span data-stu-id="1289c-115">During the 1990s, the emphasis had been mainly focused on improving internal networks using the same language and platforms.</span></span> <span data-ttu-id="1289c-116">TCP/IP wurde für diese Art von Kommunikation der Gold-Standard.</span><span class="sxs-lookup"><span data-stu-id="1289c-116">TCP/IP became the gold standard for this type of communication.</span></span>

<span data-ttu-id="1289c-117">Der Schwerpunkt liegt jedoch in Kürze auf die optimale Optimierung der Kommunikation über mehrere Plattformen hinweg, die einen sprach agnostischen Ansatz fördern.</span><span class="sxs-lookup"><span data-stu-id="1289c-117">However, the focus soon shifted to how best to optimize communication across multiple platforms promoting a language agnostic approach.</span></span> <span data-ttu-id="1289c-118">Die Dienst orientierte Architektur (Service Oriented Architecture, SOA) stellt eine Struktur für die lose Kopplung einer umfassenden Sammlung von Diensten bereit, die für eine Anwendung bereitgestellt werden können.</span><span class="sxs-lookup"><span data-stu-id="1289c-118">Service-oriented architecture (SOA) provided a structure for loosely coupling a broad collection of services that could be provided to an application.</span></span>

<span data-ttu-id="1289c-119">Die Entwicklung von *Webdiensten* ist erfolgt, nachdem alle wichtigen Plattformen auf das Internet zugreifen konnten, aber Sie konnten trotzdem nicht miteinander interagieren.</span><span class="sxs-lookup"><span data-stu-id="1289c-119">The development of *Web Services* occurred once all major platforms could access the Internet, but they still couldn’t interact with each other.</span></span> <span data-ttu-id="1289c-120">Webdienste verfügen über offene Standards und Protokolle, einschließlich:</span><span class="sxs-lookup"><span data-stu-id="1289c-120">Web services have open standards and protocols including:</span></span>

- <span data-ttu-id="1289c-121">XML zum Markieren und Codieren von Daten;</span><span class="sxs-lookup"><span data-stu-id="1289c-121">XML to tag and code data;</span></span>
- <span data-ttu-id="1289c-122">Simple Object Access-Protokoll (SOAP) zum Übertragen von Daten;</span><span class="sxs-lookup"><span data-stu-id="1289c-122">Simple Object Access Protocol (SOAP) to transfer data;</span></span>
- <span data-ttu-id="1289c-123">Web Services Definition Language (WSDL) – beschreibt den Webdienst und verbindet ihn mit der Client Anwendung. *und*</span><span class="sxs-lookup"><span data-stu-id="1289c-123">Web Services Definition Language (WSDL) – describes and connects the web service to the client application; *and*</span></span>
- <span data-ttu-id="1289c-124">Universelle Description Discovery Integration (UDDI): ermöglicht, dass Webdienste von anderen Diensten erkannt werden können.</span><span class="sxs-lookup"><span data-stu-id="1289c-124">Universal Description Discovery Integration (UDDI)- enables Web Services to be discoverable by other Services</span></span>

<span data-ttu-id="1289c-125">SOAP definiert die Regeln, nach denen verteilte Elemente einer Anwendung miteinander kommunizieren können, auch wenn Sie sich auf unterschiedlichen Plattformen befinden.</span><span class="sxs-lookup"><span data-stu-id="1289c-125">SOAP defines the rules by which distributed elements of an application can communicate with each other even if they are on different platforms.</span></span> <span data-ttu-id="1289c-126">SOAP basiert auf XML und ist daher Menschen lesbar.</span><span class="sxs-lookup"><span data-stu-id="1289c-126">SOAP is based on XML, so it's human-readable.</span></span> <span data-ttu-id="1289c-127">Das Opfer der leicht verständlichen Erstellung von SOAP ist die Größe. SOAP-Nachrichten sind größer als Nachrichten in vergleichbaren Protokollen.</span><span class="sxs-lookup"><span data-stu-id="1289c-127">The sacrifice for making SOAP easily understood is size; SOAP messages are larger than messages in comparable protocols.</span></span> <span data-ttu-id="1289c-128">SOAP wurde entwickelt, um monolithische Anwendungen in Form von mehreren Komponenten aufzulösen, ohne Sicherheit oder Kontrolle zu verlieren.</span><span class="sxs-lookup"><span data-stu-id="1289c-128">SOAP was designed to break down monolithic applications into multi-component form without losing security or control.</span></span> <span data-ttu-id="1289c-129">Daher wurde WCF so konzipiert, dass es im Gegensatz zu GrpC, das als verteiltes System begann, mit dieser Art von System funktioniert.</span><span class="sxs-lookup"><span data-stu-id="1289c-129">Therefore, WCF was designed to work with that kind of system, unlike gRPC, which began as a distributed system.</span></span> <span data-ttu-id="1289c-130">WCF hat einige dieser Einschränkungen behoben, indem er proprietäre Erweiterungs Protokolle für den SOAP-Stapel entwickelt und dokumentiert hat, aber auf Kosten der fehlenden Unterstützung von anderen Plattformen.</span><span class="sxs-lookup"><span data-stu-id="1289c-130">WCF addressed some of these limitations by developing and documenting proprietary extension protocols for the SOAP stack, but at the cost of lack of support from other platforms.</span></span>

<span data-ttu-id="1289c-131">Windows Communication Foundation ist ein Framework zum Entwickeln von Diensten.</span><span class="sxs-lookup"><span data-stu-id="1289c-131">Windows Communication Foundation is a framework for building services.</span></span> <span data-ttu-id="1289c-132">Es wurde in den frühen 2000er Jahren entworfen, um Entwicklern bei der Verwendung von frühen SOA bei der Verwaltung der komplexen arbeiten mit SOAP zu helfen.</span><span class="sxs-lookup"><span data-stu-id="1289c-132">It was designed in the early 2000s to help developers using early SOA to manage the complexities of working with SOAP.</span></span> <span data-ttu-id="1289c-133">Obwohl es nicht erforderlich ist, dass der Entwickler eigene SOAP-Protokolle schreibt, verwendet WCF weiterhin SOAP, um die Interoperabilität mit anderen Systemen zu ermöglichen.</span><span class="sxs-lookup"><span data-stu-id="1289c-133">Although it removes the requirement for the developer to write their own SOAP protocols, WCF still uses SOAP to enable interoperability with other systems.</span></span> <span data-ttu-id="1289c-134">WCF wurde auch für die Bereitstellung von Lösungen über mehrere Protokolle (http/1.1, NetTcp usw.) entwickelt.</span><span class="sxs-lookup"><span data-stu-id="1289c-134">WCF was also designed to deliver solutions across multiple protocols (HTTP/1.1, NetTCP, and so on).</span></span>

## <a name="microservices"></a><span data-ttu-id="1289c-135">Microservices</span><span class="sxs-lookup"><span data-stu-id="1289c-135">Microservices</span></span>

<span data-ttu-id="1289c-136">In microservice-Architekturen werden große Anwendungen als eine Sammlung von kleineren modularen Diensten erstellt.</span><span class="sxs-lookup"><span data-stu-id="1289c-136">In microservice architectures, large applications are built as a collection of smaller modular services.</span></span> <span data-ttu-id="1289c-137">Jede Komponente führt eine bestimmte Aufgabe oder einen bestimmten Prozess aus, und die Komponenten sind für die Interoperabilität konzipiert, können jedoch nach Bedarf isoliert werden.</span><span class="sxs-lookup"><span data-stu-id="1289c-137">Each component does a specific task or process, and components are designed to work interoperably but can be isolated as necessary.</span></span>

<span data-ttu-id="1289c-138">Zu den Vorteilen von-Diensten gehören:</span><span class="sxs-lookup"><span data-stu-id="1289c-138">Advantages to microservices include:</span></span>

- <span data-ttu-id="1289c-139">Änderungen und Upgrades können unabhängig voneinander gehandhabt werden.</span><span class="sxs-lookup"><span data-stu-id="1289c-139">Changes and upgrades can be handled independently.</span></span>
- <span data-ttu-id="1289c-140">Die Fehlerbehandlung wird effizienter, da Probleme auf bestimmte Dienste nachverfolgt werden können, die dann unabhängig von den anderen Diensten isoliert, neu erstellt, getestet und erneut bereitgestellt werden.</span><span class="sxs-lookup"><span data-stu-id="1289c-140">Error handling becomes more efficient as problems can be traced to specific services that are then isolated, rebuilt, tested, and redeployed independent of the other services.</span></span>
- <span data-ttu-id="1289c-141">Die Skalierbarkeit kann auf die spezifischen Instanzen oder Dienste und nicht auf die gesamte Anwendung beschränkt werden.</span><span class="sxs-lookup"><span data-stu-id="1289c-141">Scalability can be confined to the specific instances or services rather than the whole application.</span></span>
- <span data-ttu-id="1289c-142">Die Entwicklung kann über mehrere Teams hinweg erfolgen, und es treten weniger Probleme auf, als wenn viele Teams an einer einzelnen CodeBase arbeiten.</span><span class="sxs-lookup"><span data-stu-id="1289c-142">Development can happen across multiple teams, with less friction than occurs when many teams work on a single codebase.</span></span>

<span data-ttu-id="1289c-143">Die Umstellung auf die Erhöhung von Virtualisierung, Cloud Computing, Containern und Internet der Dinge hat zu einem kontinuierlichen Anstieg der-Dienste beigetragen.</span><span class="sxs-lookup"><span data-stu-id="1289c-143">The move towards increasing virtualization, cloud computing, containers and the Internet of Things have contributed to the ongoing rise of microservices.</span></span> <span data-ttu-id="1289c-144">Allerdings sind die-Dienste nicht ohne Ihre Herausforderungen.</span><span class="sxs-lookup"><span data-stu-id="1289c-144">However, microservices aren't without their challenges.</span></span> <span data-ttu-id="1289c-145">Die fragmentierte/dezentralisierte Infrastruktur hat bei der Kommunikation zwischen Diensten einen größeren Schwerpunkt auf den Bedarf an Einfachheit und Geschwindigkeit.</span><span class="sxs-lookup"><span data-stu-id="1289c-145">The fragmented/decentralized infrastructure placed more emphasis on the need for simplicity and speed when communicating between services.</span></span> <span data-ttu-id="1289c-146">Dies hat wiederum auf die manchmal mühsame und konforme Art von SOAP hingewiesen.</span><span class="sxs-lookup"><span data-stu-id="1289c-146">This in turn drew attention to the sometimes laborious and contorted nature of SOAP.</span></span>

<span data-ttu-id="1289c-147">In dieser Umgebung wurde GrpC gestartet, 10 Jahre nach der ersten Veröffentlichung von Microsoft WCF.</span><span class="sxs-lookup"><span data-stu-id="1289c-147">It was into this environment that gRPC was launched, 10 years after Microsoft first released WCF.</span></span> <span data-ttu-id="1289c-148">GrpC, das direkt von der internen Infrastruktur von Google (Stubby) entwickelt wurde, basiert nie auf denselben Standards und Protokollen, die die Parameter vieler früherer RPCs aufwiesen.</span><span class="sxs-lookup"><span data-stu-id="1289c-148">Evolved directly from Google’s internal infrastructure RPC (Stubby), gRPC was never based on the same standards and protocols that had informed the parameters of many earlier RPCs.</span></span> <span data-ttu-id="1289c-149">Außerdem war GrpC bisher nur auf http/2 basiert, und das ist der Grund, warum es die neuen Funktionen, die das erweiterte Transportprotokoll bereitstellt, nutzen konnte.</span><span class="sxs-lookup"><span data-stu-id="1289c-149">Additionally, gRPC was only ever based on HTTP/2 and that's why it could draw on the new capabilities that advanced transport protocol provided.</span></span> <span data-ttu-id="1289c-150">Insbesondere bidirektionales Streaming, binäres Messaging und Multiplexing.</span><span class="sxs-lookup"><span data-stu-id="1289c-150">In particular, bidirectional streaming, binary messaging, and multiplexing.</span></span>
