# <a name="microservices-addressability-and-the-service-registry"></a><span data-ttu-id="73e7e-103">Adressierbarkeit von Microservices und die Dienstregistrierung</span><span class="sxs-lookup"><span data-stu-id="73e7e-103">Microservices addressability and the service registry</span></span>

<span data-ttu-id="73e7e-104">Jeder Microservice hat einen eindeutigen Namen (URL), der zum Auflösen des Speicherorts verwendet wird.</span><span class="sxs-lookup"><span data-stu-id="73e7e-104">Each microservice has a unique name (URL) that's used to resolve its location.</span></span> <span data-ttu-id="73e7e-105">Ihr Microservice muss adressierbar sein, unabhängig davon, wo er ausgeführt wird.</span><span class="sxs-lookup"><span data-stu-id="73e7e-105">Your microservice needs to be addressable wherever it's running.</span></span> <span data-ttu-id="73e7e-106">Wenn Sie darüber nachdenken müssen, welcher Computer einen bestimmten Microservice ausführt, können schnell Fehler auftreten.</span><span class="sxs-lookup"><span data-stu-id="73e7e-106">If you have to think about which computer is running a particular microservice, things can go bad quickly.</span></span> <span data-ttu-id="73e7e-107">Ähnlich wie DNS eine URL zu einem bestimmten Computer auflöst, benötigt Ihr Microservice einen eindeutigen Namen, damit seine aktuelle Position ermittelt werden kann.</span><span class="sxs-lookup"><span data-stu-id="73e7e-107">In the same way that DNS resolves a URL to a particular computer, your microservice needs to have a unique name so that its current location is discoverable.</span></span> <span data-ttu-id="73e7e-108">Microservices benötigen adressierbare Namen, um von der Infrastruktur unabhängig zu sein, auf der sie ausgeführt werden.</span><span class="sxs-lookup"><span data-stu-id="73e7e-108">Microservices need addressable names that make them independent from the infrastructure that they're running on.</span></span> <span data-ttu-id="73e7e-109">Da eine [Dienstregistrierung](https://microservices.io/patterns/service-registry.html) vorhanden sein muss, impliziert dies, dass eine Interaktion zwischen der Bereitstellung und der Erkennung des Diensts besteht.</span><span class="sxs-lookup"><span data-stu-id="73e7e-109">This implies that there's an interaction between how your service is deployed and how it's discovered, because there needs to be a [service registry](https://microservices.io/patterns/service-registry.html).</span></span> <span data-ttu-id="73e7e-110">In gleicher Weise muss der Registrierungsdienst angeben können, wo der Dienst momentan ausgeführt wird, wenn bei einem Computer ein Fehler auftritt.</span><span class="sxs-lookup"><span data-stu-id="73e7e-110">In the same vein, when a computer fails, the registry service must be able to indicate where the service is now running.</span></span>

<span data-ttu-id="73e7e-111">Das [Muster der Dienstregistrierung](https://microservices.io/patterns/service-registry.html) ist ein Schlüsselteil der Diensterkennung.</span><span class="sxs-lookup"><span data-stu-id="73e7e-111">The [service registry pattern](https://microservices.io/patterns/service-registry.html) is a key part of service discovery.</span></span> <span data-ttu-id="73e7e-112">Die Registrierung ist eine Datenbank, die Netzwerkadressen und Dienstinstanzen enthält.</span><span class="sxs-lookup"><span data-stu-id="73e7e-112">The registry is a database containing the network locations of service instances.</span></span> <span data-ttu-id="73e7e-113">Eine Dienstregistrierung muss hochverfügbar und auf dem neuesten Stand sein.</span><span class="sxs-lookup"><span data-stu-id="73e7e-113">A service registry needs to be highly available and up-to-date.</span></span> <span data-ttu-id="73e7e-114">Clients könnten Netzwerkadressen zwischenspeichern, die sie von der Dienstregistrierung erhalten.</span><span class="sxs-lookup"><span data-stu-id="73e7e-114">Clients could cache network locations obtained from the service registry.</span></span> <span data-ttu-id="73e7e-115">Allerdings veralten diese Informationen, und Clients können die Dienstinstanzen nicht mehr ermitteln.</span><span class="sxs-lookup"><span data-stu-id="73e7e-115">However, that information eventually goes out of date and clients can no longer discover service instances.</span></span> <span data-ttu-id="73e7e-116">Folglich besteht eine Dienstregistrierung aus einem Cluster von Servern, die ein Replikationsprotokoll verwenden, um die Konsistenz zu gewährleisten.</span><span class="sxs-lookup"><span data-stu-id="73e7e-116">Consequently, a service registry consists of a cluster of servers that use a replication protocol to maintain consistency.</span></span>

<span data-ttu-id="73e7e-117">In manchen Bereitstellungsumgebungen für Microservices (auch Clusters genannt, wie in einem späteren Abschnitt erklärt wird) ist die Diensterkennung integriert.</span><span class="sxs-lookup"><span data-stu-id="73e7e-117">In some microservice deployment environments (called clusters, to be covered in a later section), service discovery is built-in.</span></span> <span data-ttu-id="73e7e-118">So kann beispielsweise eine Azure Kubernetes Service-Umgebung (AKS) die Registrierung und die Registrierungsaufhebung von Dienstinstanzen ausführen.</span><span class="sxs-lookup"><span data-stu-id="73e7e-118">For example, an Azure Container Service with Kubernetes (AKS) environment can handle service instance registration and deregistration.</span></span> <span data-ttu-id="73e7e-119">Die Umgebung führt außerdem einen Proxy auf jedem Clusterhost aus, der die Rolle eines serverseitigen Suchrouters übernimmt.</span><span class="sxs-lookup"><span data-stu-id="73e7e-119">It also runs a proxy on each cluster host that plays the role of server-side discovery router.</span></span>
