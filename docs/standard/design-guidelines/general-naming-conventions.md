<span data-ttu-id="43a74-103">In diesem Abschnitt werden allgemeine Benennungs Konventionen im Zusammenhang mit der Wort Auswahl, Richtlinien zur Verwendung von Abkürzungen und Akronymen und Empfehlungen zur Vermeidung der Verwendung von sprachspezifischen Namen beschrieben.</span><span class="sxs-lookup"><span data-stu-id="43a74-103">This section describes general naming conventions that relate to word choice, guidelines on using abbreviations and acronyms, and recommendations on how to avoid using language-specific names.</span></span>  
  
## <a name="word-choice"></a><span data-ttu-id="43a74-104">Wort Auswahl</span><span class="sxs-lookup"><span data-stu-id="43a74-104">Word Choice</span></span>  
 <span data-ttu-id="43a74-105">**✓ DO** übersichtlich Bezeichnernamen auswählen.</span><span class="sxs-lookup"><span data-stu-id="43a74-105">**✓ DO** choose easily readable identifier names.</span></span>  
  
 <span data-ttu-id="43a74-106">Beispielsweise ist eine Eigenschaft mit dem Namen `HorizontalAlignment` besser lesbar als `AlignmentHorizontal`.</span><span class="sxs-lookup"><span data-stu-id="43a74-106">For example, a property named `HorizontalAlignment` is more English-readable than `AlignmentHorizontal`.</span></span>  
  
 <span data-ttu-id="43a74-107">**✓ DO** ziehen Sie Lesbarkeit Knappheit zu finden.</span><span class="sxs-lookup"><span data-stu-id="43a74-107">**✓ DO** favor readability over brevity.</span></span>  
  
 <span data-ttu-id="43a74-108">Der Eigenschaftsname `CanScrollHorizontally` ist besser als `ScrollableX` (ein undurchsichtiger Verweis auf die X-Achse).</span><span class="sxs-lookup"><span data-stu-id="43a74-108">The property name `CanScrollHorizontally` is better than `ScrollableX` (an obscure reference to the X-axis).</span></span>  
  
 <span data-ttu-id="43a74-109">**X DO NOT** Unterstriche, Bindestriche oder andere nicht alphanumerischen Zeichen verwenden.</span><span class="sxs-lookup"><span data-stu-id="43a74-109">**X DO NOT** use underscores, hyphens, or any other nonalphanumeric characters.</span></span>  
  
 <span data-ttu-id="43a74-110">**X DO NOT** verwenden Ungarischer Notation benannt.</span><span class="sxs-lookup"><span data-stu-id="43a74-110">**X DO NOT** use Hungarian notation.</span></span>  
  
 <span data-ttu-id="43a74-111">**X AVOID** Bezeichner, die häufig mit Schlüsselwörtern in Konflikt mit der übrigen Programmiersprachen verwendet.</span><span class="sxs-lookup"><span data-stu-id="43a74-111">**X AVOID** using identifiers that conflict with keywords of widely used programming languages.</span></span>  
  
 <span data-ttu-id="43a74-112">Gemäß Regel 4 der Common Language Specification (CLS) müssen alle kompatiblen Sprachen einen Mechanismus bereitstellen, der den Zugriff auf benannte Elemente ermöglicht, die ein Schlüsselwort dieser Sprache als Bezeichner verwenden.</span><span class="sxs-lookup"><span data-stu-id="43a74-112">According to Rule 4 of the Common Language Specification (CLS), all compliant languages must provide a mechanism that allows access to named items that use a keyword of that language as an identifier.</span></span> <span data-ttu-id="43a74-113">C#Beispielsweise wird in diesem Fall das @-Zeichen als Escapezeichen verwendet.</span><span class="sxs-lookup"><span data-stu-id="43a74-113">C#, for example, uses the @ sign as an escape mechanism in this case.</span></span> <span data-ttu-id="43a74-114">Es ist jedoch immer noch eine gute Idee, gängige Schlüsselwörter zu vermeiden, da es weitaus schwieriger ist, eine Methode mit der Escapesequenz zu verwenden, als Sie ohne Sie.</span><span class="sxs-lookup"><span data-stu-id="43a74-114">However, it is still a good idea to avoid common keywords because it is much more difficult to use a method with the escape sequence than one without it.</span></span>  
  
## <a name="using-abbreviations-and-acronyms"></a><span data-ttu-id="43a74-115">Verwenden von Abkürzungen und Akronymen</span><span class="sxs-lookup"><span data-stu-id="43a74-115">Using Abbreviations and Acronyms</span></span>  
 <span data-ttu-id="43a74-116">**X DO NOT** Abkürzungen oder Kontraktionen als Teil des Bezeichnernamen verwenden.</span><span class="sxs-lookup"><span data-stu-id="43a74-116">**X DO NOT** use abbreviations or contractions as part of identifier names.</span></span>  
  
 <span data-ttu-id="43a74-117">Verwenden Sie z. b. `GetWindow` anstelle von `GetWin`.</span><span class="sxs-lookup"><span data-stu-id="43a74-117">For example, use `GetWindow` rather than `GetWin`.</span></span>  
  
 <span data-ttu-id="43a74-118">**X DO NOT** verwenden Sie Akronyme, die nicht weit verbreitete und sogar, wenn sie, falls erforderlich sind.</span><span class="sxs-lookup"><span data-stu-id="43a74-118">**X DO NOT** use any acronyms that are not widely accepted, and even if they are, only when necessary.</span></span>

 <span data-ttu-id="43a74-120">**✓ DO** verwenden semantisch interessante Namen anstelle der Language-spezifische Schlüsselwörter für Typnamen.</span><span class="sxs-lookup"><span data-stu-id="43a74-120">**✓ DO** use semantically interesting names rather than language-specific keywords for type names.</span></span>  
  
 <span data-ttu-id="43a74-121">Beispielsweise ist `GetLength` ein besserer Name als `GetInt`.</span><span class="sxs-lookup"><span data-stu-id="43a74-121">For example, `GetLength` is a better name than `GetInt`.</span></span>  
  
 <span data-ttu-id="43a74-122">**✓ DO** verwenden Sie eine generische CLR-Typnamen, anstatt einen sprachspezifischen Namen, in den seltenen Fällen, wenn ein Bezeichner keine semantische Bedeutung außer seinem Typ aufweist.</span><span class="sxs-lookup"><span data-stu-id="43a74-122">**✓ DO** use a generic CLR type name, rather than a language-specific name, in the rare cases when an identifier has no semantic meaning beyond its type.</span></span>

 <span data-ttu-id="43a74-188">Dadurch wird die Beziehung zwischen den APIs hervorgehoben.</span><span class="sxs-lookup"><span data-stu-id="43a74-188">This helps to highlight the relationship between the APIs.</span></span>  
  
 <span data-ttu-id="43a74-189">**✓ DO** bevorzugen Sie ein Suffix anstelle ein Präfix an, dass eine neue Version einer vorhandenen API hinzufügen.</span><span class="sxs-lookup"><span data-stu-id="43a74-189">**✓ DO** prefer adding a suffix rather than a prefix to indicate a new version of an existing API.</span></span>  
  
 <span data-ttu-id="43a74-190">Dadurch wird die Ermittlung beim Durchsuchen der Dokumentation oder bei Verwendung von IntelliSense unterstützt.</span><span class="sxs-lookup"><span data-stu-id="43a74-190">This will assist discovery when browsing documentation, or using IntelliSense.</span></span> <span data-ttu-id="43a74-191">Die alte Version der API wird in der Nähe der neuen APIs organisiert, da die meisten Browser und IntelliSense Bezeichner in alphabetischer Reihenfolge anzeigen.</span><span class="sxs-lookup"><span data-stu-id="43a74-191">The old version of the API will be organized close to the new APIs, because most browsers and IntelliSense show identifiers in alphabetical order.</span></span>
