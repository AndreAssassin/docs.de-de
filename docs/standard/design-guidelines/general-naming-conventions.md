# <a name="general-naming-conventions"></a><span data-ttu-id="0ee53-102">Allgemeine Benennungskonventionen</span><span class="sxs-lookup"><span data-stu-id="0ee53-102">General Naming Conventions</span></span>
<span data-ttu-id="0ee53-103">Dieser Abschnitt beschreibt allgemeine Benennungskonventionen, die im Zusammenhang mit der Auswahl der Wörter, die Richtlinien zum Verwenden von Abkürzungen und Akronyme und Empfehlungen zum Vermeiden Sie sprachspezifische Namen.</span><span class="sxs-lookup"><span data-stu-id="0ee53-103">This section describes general naming conventions that relate to word choice, guidelines on using abbreviations and acronyms, and recommendations on how to avoid using language-specific names.</span></span>  
  
## <a name="word-choice"></a><span data-ttu-id="0ee53-104">Word-Auswahl</span><span class="sxs-lookup"><span data-stu-id="0ee53-104">Word Choice</span></span>  
 <span data-ttu-id="0ee53-105">**✓ DO** übersichtlich Bezeichnernamen auswählen.</span><span class="sxs-lookup"><span data-stu-id="0ee53-105">**✓ DO** choose easily readable identifier names.</span></span>  
  
 <span data-ttu-id="0ee53-106">Z. B. eine Eigenschaft namens `HorizontalAlignment` ist Englisch-lesbarer als `AlignmentHorizontal`.</span><span class="sxs-lookup"><span data-stu-id="0ee53-106">For example, a property named `HorizontalAlignment` is more English-readable than `AlignmentHorizontal`.</span></span>  
  
 <span data-ttu-id="0ee53-107">**✓ DO** ziehen Sie Lesbarkeit Knappheit zu finden.</span><span class="sxs-lookup"><span data-stu-id="0ee53-107">**✓ DO** favor readability over brevity.</span></span>  
  
 <span data-ttu-id="0ee53-108">Der Eigenschaftsname `CanScrollHorizontally` ist besser als `ScrollableX` (eine ungewöhnliche Verweis auf die X-Achse).</span><span class="sxs-lookup"><span data-stu-id="0ee53-108">The property name `CanScrollHorizontally` is better than `ScrollableX` (an obscure reference to the X-axis).</span></span>  
  
 <span data-ttu-id="0ee53-109">**X DO NOT** Unterstriche, Bindestriche oder andere nicht alphanumerischen Zeichen verwenden.</span><span class="sxs-lookup"><span data-stu-id="0ee53-109">**X DO NOT** use underscores, hyphens, or any other nonalphanumeric characters.</span></span>  
  
 <span data-ttu-id="0ee53-110">**X DO NOT** verwenden Ungarischer Notation benannt.</span><span class="sxs-lookup"><span data-stu-id="0ee53-110">**X DO NOT** use Hungarian notation.</span></span>  
  
 <span data-ttu-id="0ee53-111">**X AVOID** Bezeichner, die häufig mit Schlüsselwörtern in Konflikt mit der übrigen Programmiersprachen verwendet.</span><span class="sxs-lookup"><span data-stu-id="0ee53-111">**X AVOID** using identifiers that conflict with keywords of widely used programming languages.</span></span>  
  
 <span data-ttu-id="0ee53-112">Entsprechend der Regel 4, von der Common Language Specification (CLS) müssen alle kompatible Sprachen einen Mechanismus bereitstellen, der Zugriff auf benannte Elemente ermöglicht, die ein Schlüsselwort der Sprache als Bezeichner verwenden.</span><span class="sxs-lookup"><span data-stu-id="0ee53-112">According to Rule 4 of the Common Language Specification (CLS), all compliant languages must provide a mechanism that allows access to named items that use a keyword of that language as an identifier.</span></span> <span data-ttu-id="0ee53-113">C# verwendet beispielsweise das @-Zeichen als Escapesequenz Mechanismus in diesem Fall.</span><span class="sxs-lookup"><span data-stu-id="0ee53-113">C#, for example, uses the @ sign as an escape mechanism in this case.</span></span> <span data-ttu-id="0ee53-114">Allerdings ist es immer noch eine gute Idee, häufige oder gemeinsame Schlüsselwörter zu vermeiden, da es sehr viel schwieriger, eine Methode mit der Escapesequenz als ein ohne diese zu verwenden ist.</span><span class="sxs-lookup"><span data-stu-id="0ee53-114">However, it is still a good idea to avoid common keywords because it is much more difficult to use a method with the escape sequence than one without it.</span></span>  
  
## <a name="using-abbreviations-and-acronyms"></a><span data-ttu-id="0ee53-115">Verwenden von Abkürzungen und Akronyme</span><span class="sxs-lookup"><span data-stu-id="0ee53-115">Using Abbreviations and Acronyms</span></span>  
 <span data-ttu-id="0ee53-116">**X DO NOT** Abkürzungen oder Kontraktionen als Teil des Bezeichnernamen verwenden.</span><span class="sxs-lookup"><span data-stu-id="0ee53-116">**X DO NOT** use abbreviations or contractions as part of identifier names.</span></span>  
  
 <span data-ttu-id="0ee53-117">Verwenden Sie z. B. `GetWindow` statt `GetWin`.</span><span class="sxs-lookup"><span data-stu-id="0ee53-117">For example, use `GetWindow` rather than `GetWin`.</span></span>  
  
 <span data-ttu-id="0ee53-118">**X DO NOT** verwenden Sie Akronyme, die nicht weit verbreitete und sogar, wenn sie, falls erforderlich sind.</span><span class="sxs-lookup"><span data-stu-id="0ee53-118">**X DO NOT** use any acronyms that are not widely accepted, and even if they are, only when necessary.</span></span>

 <span data-ttu-id="0ee53-120">**✓ DO** verwenden semantisch interessante Namen anstelle der Language-spezifische Schlüsselwörter für Typnamen.</span><span class="sxs-lookup"><span data-stu-id="0ee53-120">**✓ DO** use semantically interesting names rather than language-specific keywords for type names.</span></span>  
  
 <span data-ttu-id="0ee53-121">Z. B. `GetLength` ist ein besserer Name als `GetInt`.</span><span class="sxs-lookup"><span data-stu-id="0ee53-121">For example, `GetLength` is a better name than `GetInt`.</span></span>  
  
 <span data-ttu-id="0ee53-122">**✓ DO** verwenden Sie eine generische CLR-Typnamen, anstatt einen sprachspezifischen Namen, in den seltenen Fällen, wenn ein Bezeichner keine semantische Bedeutung außer seinem Typ aufweist.</span><span class="sxs-lookup"><span data-stu-id="0ee53-122">**✓ DO** use a generic CLR type name, rather than a language-specific name, in the rare cases when an identifier has no semantic meaning beyond its type.</span></span>

## <a name="naming-new-versions-of-existing-apis"></a><span data-ttu-id="0ee53-186">Benennen die neue Versionen der vorhandenen APIs</span><span class="sxs-lookup"><span data-stu-id="0ee53-186">Naming New Versions of Existing APIs</span></span>  
 <span data-ttu-id="0ee53-187">**✓ DO** einen ähnlichen Namen wie der alte-API verwenden, wenn neue Versionen einer vorhandenen API zu erstellen.</span><span class="sxs-lookup"><span data-stu-id="0ee53-187">**✓ DO** use a name similar to the old API when creating new versions of an existing API.</span></span>  
  
 <span data-ttu-id="0ee53-188">Dadurch wird um die Beziehung zwischen der APIs zu markieren.</span><span class="sxs-lookup"><span data-stu-id="0ee53-188">This helps to highlight the relationship between the APIs.</span></span>  
  
 <span data-ttu-id="0ee53-189">**✓ DO** bevorzugen Sie ein Suffix anstelle ein Präfix an, dass eine neue Version einer vorhandenen API hinzufügen.</span><span class="sxs-lookup"><span data-stu-id="0ee53-189">**✓ DO** prefer adding a suffix rather than a prefix to indicate a new version of an existing API.</span></span>  
  
 <span data-ttu-id="0ee53-190">Dies hilft beim Durchsuchen der Dokumentation, die Ermittlung oder mithilfe von IntelliSense.</span><span class="sxs-lookup"><span data-stu-id="0ee53-190">This will assist discovery when browsing documentation, or using IntelliSense.</span></span> <span data-ttu-id="0ee53-191">Die alte Version der API wird in der Nähe der neuen APIs, organisiert werden, da die meisten Browser und IntelliSense Bezeichner in alphabetischer Reihenfolge angezeigt.</span><span class="sxs-lookup"><span data-stu-id="0ee53-191">The old version of the API will be organized close to the new APIs, because most browsers and IntelliSense show identifiers in alphabetical order.</span></span>  
  
 <span data-ttu-id="0ee53-192">**✓ CONSIDER** mithilfe eines brandneuen, jedoch aussagekräftigen-Bezeichners, anstatt ein Suffix oder Präfix.</span><span class="sxs-lookup"><span data-stu-id="0ee53-192">**✓ CONSIDER** using a brand new, but meaningful identifier, instead of adding a suffix or a prefix.</span></span>  
  
 <span data-ttu-id="0ee53-193">**✓ DO** ein numerisches Suffix verwenden, um eine neue Version einer vorhandenen API anzugeben, insbesondere, wenn der vorhandene Name der API der einzige Name, die sinnvoll ist (d. h., wenn es sich um einen Industriestandard ist) und wenn jeder sinnvolle hinzufügen Suffix (oder Ändern des Namens) keine app Ropriate-Option.</span><span class="sxs-lookup"><span data-stu-id="0ee53-193">**✓ DO** use a numeric suffix to indicate a new version of an existing API, particularly if the existing name of the API is the only name that makes sense (i.e., if it is an industry standard) and if adding any meaningful suffix (or changing the name) is not an appropriate option.</span></span>  
  
 <span data-ttu-id="0ee53-194">**X DO NOT** verwenden, die "Ex" (oder ähnlich)-Suffix für einen Bezeichner zur Unterscheidung von einer früheren Version der gleichen-API.</span><span class="sxs-lookup"><span data-stu-id="0ee53-194">**X DO NOT** use the "Ex" (or a similar) suffix for an identifier to distinguish it from an earlier version of the same API.</span></span>  
  
 <span data-ttu-id="0ee53-195">**✓ DO** verwenden Sie das Suffix "64", bei der Einführung von Versionen von APIs, die für eine 64-Bit-Ganzzahl (eine lange ganze Zahl) anstelle von 32-Bit-Ganzzahl ausgeführt werden.</span><span class="sxs-lookup"><span data-stu-id="0ee53-195">**✓ DO** use the "64" suffix when introducing versions of APIs that operate on a 64-bit integer (a long integer) instead of a 32-bit integer.</span></span> <span data-ttu-id="0ee53-196">Sie müssen sich nur auf diesen Ansatz verwenden, wenn die vorhandene 32-Bit-API vorhanden ist. Lassen Sie es für neue APIs mit nur einer 64-Bit-Version.</span><span class="sxs-lookup"><span data-stu-id="0ee53-196">You only need to take this approach when the existing 32-bit API exists; don’t do it for brand new APIs with only a 64-bit version.</span></span>
