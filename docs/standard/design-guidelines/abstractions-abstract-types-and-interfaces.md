# <a name="abstractions-abstract-types-and-interfaces"></a><span data-ttu-id="03c74-102">Abstraktionen (abstrakte Typen und Schnittstellen)</span><span class="sxs-lookup"><span data-stu-id="03c74-102">Abstractions (Abstract Types and Interfaces)</span></span>
<span data-ttu-id="03c74-103">Eine Abstraktion ist ein Typ, der beschreibt einen Vertrag, aber keine vollständige Implementierung des Vertrags bereitstellt.</span><span class="sxs-lookup"><span data-stu-id="03c74-103">An abstraction is a type that describes a contract but does not provide a full implementation of the contract.</span></span> <span data-ttu-id="03c74-104">Abstraktionen sind in der Regel als abstrakten Klassen oder Schnittstellen implementiert, und sie verfügen über einen genau definierten Satz von Dokumentation beschreiben die erforderliche Semantik der Typen, die den Vertrag implementiert.</span><span class="sxs-lookup"><span data-stu-id="03c74-104">Abstractions are usually implemented as abstract classes or interfaces, and they come with a well-defined set of reference documentation describing the required semantics of the types implementing the contract.</span></span> <span data-ttu-id="03c74-105">Einige der wichtigsten Abstraktionen in .NET Framework sind <xref:System.IO.Stream>, <xref:System.Collections.Generic.IEnumerable%601>, und <xref:System.Object>.</span><span class="sxs-lookup"><span data-stu-id="03c74-105">Some of the most important abstractions in the .NET Framework include <xref:System.IO.Stream>, <xref:System.Collections.Generic.IEnumerable%601>, and <xref:System.Object>.</span></span>  
  
 <span data-ttu-id="03c74-106">Sie können Frameworks erweitern, indem Sie implementieren einen konkreten Typ, der den Vertrag, einer Abstraktion unterstützt und verwenden dieses konkreten Typ, mit dem Framework APIs nutzen (Ausführen auf) die Abstraktion.</span><span class="sxs-lookup"><span data-stu-id="03c74-106">You can extend frameworks by implementing a concrete type that supports the contract of an abstraction and using this concrete type with framework APIs consuming (operating on) the abstraction.</span></span>  
  
 <span data-ttu-id="03c74-107">Eine sinnvolle und nützliche Abstraktion, die den Test der Zeit standhalten ist sehr schwer zu entwerfen.</span><span class="sxs-lookup"><span data-stu-id="03c74-107">A meaningful and useful abstraction that is able to withstand the test of time is very difficult to design.</span></span> <span data-ttu-id="03c74-108">Die wichtigsten Probleme erhält die richtige Gruppe von Elementen, die nicht mehr und nicht weniger.</span><span class="sxs-lookup"><span data-stu-id="03c74-108">The main difficulty is getting the right set of members, no more and no fewer.</span></span> <span data-ttu-id="03c74-109">Wenn eine Abstraktion über zu viele Member enthält, wird es schwierig oder gar als unmöglich zu implementieren.</span><span class="sxs-lookup"><span data-stu-id="03c74-109">If an abstraction has too many members, it becomes difficult or even impossible to implement.</span></span> <span data-ttu-id="03c74-110">Wenn sie zu wenige Elemente für die zugesicherten Funktionen verfügt, wird es nutzlos für viele interessante Szenarios.</span><span class="sxs-lookup"><span data-stu-id="03c74-110">If it has too few members for the promised functionality, it becomes useless in many interesting scenarios.</span></span>  
  
 <span data-ttu-id="03c74-111">Zu viele Abstraktionen, die in einem Framework beeinflussen auch negativ auf die Nutzbarkeit von Framework.</span><span class="sxs-lookup"><span data-stu-id="03c74-111">Too many abstractions in a framework also negatively affect usability of the framework.</span></span> <span data-ttu-id="03c74-112">Es ist oft schwierig zu verstehen, eine Abstraktion ohne zu verstehen, wie es sich in den besseren Überblick über die konkreten Implementierungen und die APIs, die für die Abstraktion einfügt.</span><span class="sxs-lookup"><span data-stu-id="03c74-112">It is often quite difficult to understand an abstraction without understanding how it fits into the larger picture of the concrete implementations and the APIs operating on the abstraction.</span></span> <span data-ttu-id="03c74-113">Darüber hinaus sind Namen von Abstraktionen und deren Member notwendigerweise abstrakt, wodurch oft sie kryptische und Zufall ohne erste Kenntnis der breiteren Kontexts ihrer Verwendung.</span><span class="sxs-lookup"><span data-stu-id="03c74-113">Also, names of abstractions and their members are necessarily abstract, which often makes them cryptic and unapproachable without first understanding the broader context of their usage.</span></span>  
  
 <span data-ttu-id="03c74-114">Abstraktionen bieten jedoch äußerst leistungsfähige Erweiterbarkeit der Lösung, die die andere geeignete Erweiterungsmechanismen häufig entsprechen nicht möglich.</span><span class="sxs-lookup"><span data-stu-id="03c74-114">However, abstractions provide extremely powerful extensibility that the other extensibility mechanisms cannot often match.</span></span> <span data-ttu-id="03c74-115">Sie sind das Herzstück von viele architektonische Muster, wie z. B.-Plug-ins IoC Inversion of Control (IoC), Pipelines und So weiter.</span><span class="sxs-lookup"><span data-stu-id="03c74-115">They are at the core of many architectural patterns, such as plug-ins, inversion of control (IoC), pipelines, and so on.</span></span> <span data-ttu-id="03c74-116">Sie sind auch äußerst wichtig für testbarkeit des Frameworks.</span><span class="sxs-lookup"><span data-stu-id="03c74-116">They are also extremely important for testability of frameworks.</span></span> <span data-ttu-id="03c74-117">Gute Abstraktionen ermöglichen die stub-out hohe Abhängigkeiten in Komponententests.</span><span class="sxs-lookup"><span data-stu-id="03c74-117">Good abstractions make it possible to stub out heavy dependencies for the purpose of unit testing.</span></span> <span data-ttu-id="03c74-118">Zusammenfassend lässt sich sagen sind Abstraktionen für die moderne, objektorientierte Frameworks der gefragtesten reichhaltigen verantwortlich.</span><span class="sxs-lookup"><span data-stu-id="03c74-118">In summary, abstractions are responsible for the sought-after richness of the modern object-oriented frameworks.</span></span>  
  
 <span data-ttu-id="03c74-119">**X DO NOT** stellen Abstraktionen bereit, es sei denn, sie getestet werden, indem Sie mehrere konkrete Implementierungen und APIs, nutzen die Abstraktionen entwickeln.</span><span class="sxs-lookup"><span data-stu-id="03c74-119">**X DO NOT** provide abstractions unless they are tested by developing several concrete implementations and APIs consuming the abstractions.</span></span>  
  
 <span data-ttu-id="03c74-120">**✓ DO** wählen Sie beim Entwerfen einer Abstraktion sorgfältig zwischen eine abstrakte Klasse und einer Schnittstelle.</span><span class="sxs-lookup"><span data-stu-id="03c74-120">**✓ DO** choose carefully between an abstract class and an interface when designing an abstraction.</span></span>  
  
 <span data-ttu-id="03c74-121">**✓ CONSIDER** Verweis Tests für konkrete Implementierungen von Abstraktionen bereitstellen.</span><span class="sxs-lookup"><span data-stu-id="03c74-121">**✓ CONSIDER** providing reference tests for concrete implementations of abstractions.</span></span> <span data-ttu-id="03c74-122">Solche Tests sollten Benutzern zu prüfen, ob ihre Implementierungen ordnungsgemäß den Vertrag implementieren.</span><span class="sxs-lookup"><span data-stu-id="03c74-122">Such tests should allow users to test whether their implementations correctly implement the contract.</span></span>
