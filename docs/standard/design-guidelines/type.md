# <a name="type-design-guidelines"></a><span data-ttu-id="c7ead-102">Richtlinien für den Entwurf von Typen</span><span class="sxs-lookup"><span data-stu-id="c7ead-102">Type Design Guidelines</span></span>
<span data-ttu-id="c7ead-103">Aus der CLR-Sicht gibt es nur zwei Kategorien von Typen – Verweis Typen und Werttypen – aber zum Zweck der Erörterung des frameworkentwurfs werden Typen in weitere logische Gruppen unterteilt, die jeweils über eigene spezifische Entwurfs Regeln verfügen.</span><span class="sxs-lookup"><span data-stu-id="c7ead-103">From the CLR perspective, there are only two categories of types—reference types and value types—but for the purpose of a discussion about framework design, we divide types into more logical groups, each with its own specific design rules.</span></span>

 <span data-ttu-id="c7ead-104">Klassen sind der allgemeine Fall von Verweis Typen.</span><span class="sxs-lookup"><span data-stu-id="c7ead-104">Classes are the general case of reference types.</span></span> <span data-ttu-id="c7ead-105">Sie bilden die Mehrzahl der Typen in den meisten Frameworks.</span><span class="sxs-lookup"><span data-stu-id="c7ead-105">They make up the bulk of types in the majority of frameworks.</span></span> <span data-ttu-id="c7ead-106">Klassen verdanken ihre Beliebtheit dem umfangreichen Satz von objektorientierten Funktionen, die Sie unterstützen, und ihrer allgemeinen Anwendbarkeit.</span><span class="sxs-lookup"><span data-stu-id="c7ead-106">Classes owe their popularity to the rich set of object-oriented features they support and to their general applicability.</span></span> <span data-ttu-id="c7ead-107">Basisklassen und abstrakte Klassen sind spezielle logische Gruppen in Bezug auf die Erweiterbarkeit.</span><span class="sxs-lookup"><span data-stu-id="c7ead-107">Base classes and abstract classes are special logical groups related to extensibility.</span></span>

 <span data-ttu-id="c7ead-108">Schnittstellen sind Typen, die durch Verweis Typen und Werttypen implementiert werden können.</span><span class="sxs-lookup"><span data-stu-id="c7ead-108">Interfaces are types that can be implemented by both reference types and value types.</span></span> <span data-ttu-id="c7ead-109">Sie können daher als Stämme von polymorphen Hierarchien von Verweis Typen und Werttypen fungieren.</span><span class="sxs-lookup"><span data-stu-id="c7ead-109">They can thus serve as roots of polymorphic hierarchies of reference types and value types.</span></span> <span data-ttu-id="c7ead-110">Außerdem können Schnittstellen verwendet werden, um die mehrfache Vererbung zu simulieren, die von der CLR nicht nativ unterstützt wird.</span><span class="sxs-lookup"><span data-stu-id="c7ead-110">In addition, interfaces can be used to simulate multiple inheritance, which is not natively supported by the CLR.</span></span>

 <span data-ttu-id="c7ead-111">Strukturen sind der allgemeine Fall von Werttypen und sollten für kleine, einfache Typen reserviert werden, ähnlich wie bei sprach primitiven.</span><span class="sxs-lookup"><span data-stu-id="c7ead-111">Structs are the general case of value types and should be reserved for small, simple types, similar to language primitives.</span></span>

 <span data-ttu-id="c7ead-112">Enumerationswerte sind ein Sonderfall von Werttypen, die verwendet werden, um kurze Sätze von Werten zu definieren, z. b. Wochentage, Konsolen Farben usw.</span><span class="sxs-lookup"><span data-stu-id="c7ead-112">Enums are a special case of value types used to define short sets of values, such as days of the week, console colors, and so on.</span></span>

 <span data-ttu-id="c7ead-113">Statische Klassen sind Typen, die als Container für statische Member gedacht sind.</span><span class="sxs-lookup"><span data-stu-id="c7ead-113">Static classes are types intended to be containers for static members.</span></span> <span data-ttu-id="c7ead-114">Sie werden häufig verwendet, um Verknüpfungen zu anderen Vorgängen bereitzustellen.</span><span class="sxs-lookup"><span data-stu-id="c7ead-114">They are commonly used to provide shortcuts to other operations.</span></span>

 <span data-ttu-id="c7ead-115">Delegaten, Ausnahmen, Attribute, Arrays und Auflistungen sind Sonderfälle von Referenztypen, die für bestimmte Verwendungszwecke bestimmt sind, und Richtlinien für Ihren Entwurf und ihre Verwendung werden an anderer Stelle dieses Buchs behandelt.</span><span class="sxs-lookup"><span data-stu-id="c7ead-115">Delegates, exceptions, attributes, arrays, and collections are all special cases of reference types intended for specific uses, and guidelines for their design and usage are discussed elsewhere in this book.</span></span>

 <span data-ttu-id="c7ead-116">✔️ Stellen Sie sicher, dass jeder Typ eine klar definierte Gruppe verwandter Member ist, nicht nur eine zufällige Auflistung von nicht verknüpften Funktionen.</span><span class="sxs-lookup"><span data-stu-id="c7ead-116">✔️ DO ensure that each type is a well-defined set of related members, not just a random collection of unrelated functionality.</span></span>
