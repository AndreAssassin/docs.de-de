 <span data-ttu-id="91a26-174">Dies sollten Sie nur tun, wenn Sie den gesamten Codepfad als Sonderfall verwenden, nicht nur ein Array erstellen und die allgemeinere Methode aufrufen.</span><span class="sxs-lookup"><span data-stu-id="91a26-174">You should only do this if you are going to special-case the entire code path, not just create an array and call the more general method.</span></span>  
  
 <span data-ttu-id="91a26-175">**✓ DO** Beachten Sie, dass Null kann als ein Params-Argument übergeben werden.</span><span class="sxs-lookup"><span data-stu-id="91a26-175">**✓ DO** be aware that null could be passed as a params array argument.</span></span>

 <span data-ttu-id="91a26-177">**X DO NOT** verwenden die `varargs` Methoden, sogenannten mit den Auslassungspunkten.</span><span class="sxs-lookup"><span data-stu-id="91a26-177">**X DO NOT** use the `varargs` methods, otherwise known as the ellipsis.</span></span>

### <a name="pointer-parameters"></a><span data-ttu-id="91a26-180">Zeigerparameter</span><span class="sxs-lookup"><span data-stu-id="91a26-180">Pointer Parameters</span></span>  
 <span data-ttu-id="91a26-181">Im Allgemeinen sollten Zeiger nicht in der öffentlichen Oberfläche eines gut entworfenen verwalteten Code-Frameworks angezeigt werden.</span><span class="sxs-lookup"><span data-stu-id="91a26-181">In general, pointers should not appear in the public surface area of a well-designed managed code framework.</span></span> <span data-ttu-id="91a26-182">In den meisten Fällen sollten Zeiger gekapselt werden.</span><span class="sxs-lookup"><span data-stu-id="91a26-182">Most of the time, pointers should be encapsulated.</span></span> <span data-ttu-id="91a26-183">In einigen Fällen sind Zeiger aber aus Interoperabilitäts Gründen erforderlich, und die Verwendung von Zeigern in solchen Fällen ist angemessen.</span><span class="sxs-lookup"><span data-stu-id="91a26-183">However, in some cases pointers are required for interoperability reasons, and using pointers in such cases is appropriate.</span></span>  
  
 <span data-ttu-id="91a26-184">**✓ DO** bieten eine Alternative für ein Element, das ein Zeigerargument akzeptiert, da Zeiger nicht CLS-kompatibel sind.</span><span class="sxs-lookup"><span data-stu-id="91a26-184">**✓ DO** provide an alternative for any member that takes a pointer argument, because pointers are not CLS-compliant.</span></span>  
  
 <span data-ttu-id="91a26-185">**X AVOID** auf diese Weise die aufwändige Prüfung von Zeigerargumente Argument.</span><span class="sxs-lookup"><span data-stu-id="91a26-185">**X AVOID** doing expensive argument checking of pointer arguments.</span></span>  
  
 <span data-ttu-id="91a26-186">**✓ DO** Konventionen gemeinsamen Zeiger-bezogene beim Entwerfen der Member mit Zeigern.</span><span class="sxs-lookup"><span data-stu-id="91a26-186">**✓ DO** follow common pointer-related conventions when designing members with pointers.</span></span>  
  
 <span data-ttu-id="91a26-187">Beispielsweise ist es nicht erforderlich, den Start Index zu übergeben, da einfache Zeigerarithmetik verwendet werden kann, um das gleiche Ergebnis zu erzielen.</span><span class="sxs-lookup"><span data-stu-id="91a26-187">For example, there is no need to pass the start index, because simple pointer arithmetic can be used to accomplish the same result.</span></span>
