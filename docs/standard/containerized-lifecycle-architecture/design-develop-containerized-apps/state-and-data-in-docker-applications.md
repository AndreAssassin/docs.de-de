<span data-ttu-id="8af81-109">Über den Docker-Host als [Docker-Volumes](https://docs.docker.com/engine/admin/volumes/):</span><span class="sxs-lookup"><span data-stu-id="8af81-109">From the Docker host, as [Docker Volumes](https://docs.docker.com/engine/admin/volumes/):</span></span>

- <span data-ttu-id="8af81-111">**Binden Sie Bereitstellungen** können zu einem beliebigen Ordner in das Dateisystem Host zuordnen, damit Zugriff kann nicht aus einem Docker-Prozess gesteuert werden und kann ein Sicherheitsrisiko darstellen, wie ein Container Zugriff auf vertrauliche OS-Ordner konnte.</span><span class="sxs-lookup"><span data-stu-id="8af81-111">**Bind mounts** can map to any folder in the host filesystem, so access can't be controlled from a Docker process and can pose a security risk as a container could access sensitive OS folders.</span></span>

<span data-ttu-id="8af81-113">Über den Remotespeicher:</span><span class="sxs-lookup"><span data-stu-id="8af81-113">From remote storage:</span></span>

- <span data-ttu-id="8af81-114">[Azure-Speicher](https://azure.microsoft.com/documentation/services/storage/) bietet geografisch verteilbaren Speicher, eine gute Lösung für langfristige Persistenz für Container bereitstellen.</span><span class="sxs-lookup"><span data-stu-id="8af81-114">[Azure Storage](https://azure.microsoft.com/documentation/services/storage/) provides geo-distributable storage, providing a good long-term persistence solution for containers.</span></span>

- <span data-ttu-id="8af81-115">Relationale Remotedatenbanken wie [Azure SQL-Datenbank](https://azure.microsoft.com/services/sql-database/), NoSQL-Datenbanken wie [Azure Cosmos DB](https://docs.microsoft.com/azure/cosmos-db/introduction), oder zwischenspeicherdienste wie [Redis](https://redis.io/).</span><span class="sxs-lookup"><span data-stu-id="8af81-115">Remote relational databases like [Azure SQL Database](https://azure.microsoft.com/services/sql-database/), NoSQL databases like [Azure Cosmos DB](https://docs.microsoft.com/azure/cosmos-db/introduction), or cache services like [Redis](https://redis.io/).</span></span>

<span data-ttu-id="8af81-116">Über den Docker-Container:</span><span class="sxs-lookup"><span data-stu-id="8af81-116">From the Docker container:</span></span>

<span data-ttu-id="8af81-124">Im folgenden finden weiteren Details zu diesen Optionen.</span><span class="sxs-lookup"><span data-stu-id="8af81-124">The following provides additional detail about these options.</span></span>

<span data-ttu-id="8af81-125">**Volumes** sind Verzeichnisse, die vom Hostbetriebssystem den Verzeichnissen in Containern zugeordnet werden.</span><span class="sxs-lookup"><span data-stu-id="8af81-125">**Volumes** are directories mapped from the host OS to directories in containers.</span></span> <span data-ttu-id="8af81-126">Wenn Code im Container Zugriff auf das Verzeichnis hat, ist dies ein Zugriff auf ein Verzeichnis auf dem Hostbetriebssystem.</span><span class="sxs-lookup"><span data-stu-id="8af81-126">When code in the container has access to the directory, that access is actually to a directory on the host OS.</span></span> <span data-ttu-id="8af81-127">Dieses Verzeichnis ist nicht an die Lebensdauer des Containers selbst gebunden, und es wird von Docker verwaltet und von der Kernfunktionalität des Hostcomputers isoliert.</span><span class="sxs-lookup"><span data-stu-id="8af81-127">This directory is not tied to the lifetime of the container itself, and the directory is managed by Docker and isolated from the core functionality of the host machine.</span></span> <span data-ttu-id="8af81-128">Deshalb sind diese Datenvolumes dazu entworfen, Daten unabhängig von der Lebensdauer des Containers beizubehalten.</span><span class="sxs-lookup"><span data-stu-id="8af81-128">Thus, data volumes are designed to persist data independently of the life of the container.</span></span> <span data-ttu-id="8af81-129">Wenn Sie einen Container oder ein Image aus dem Docker-Host löschen, werden die persistenten Daten im Datenvolume nicht gelöscht.</span><span class="sxs-lookup"><span data-stu-id="8af81-129">If you delete a container or an image from the Docker host, the data persisted in the data volume is not deleted.</span></span>
