<span data-ttu-id="fc199-107">Für jede F#-Codedatei kann eine *Signaturdatei*vorhanden sein. |<span data-ttu-id="fc199-122">Attribut</span><span class="sxs-lookup"><span data-stu-id="fc199-122">Attribute</span></span>|<span data-ttu-id="fc199-123">Beschreibung</span><span class="sxs-lookup"><span data-stu-id="fc199-123">Description</span></span>|

|`[<Sealed>]`|<span data-ttu-id="fc199-124">Für einen Typ, der über keine abstrakten Member verfügt oder nicht erweitert werden darf.</span><span class="sxs-lookup"><span data-stu-id="fc199-124">For a type that has no abstract members, or that should not be extended.</span></span>|
|`[<Interface>]`|<span data-ttu-id="fc199-125">Für einen Typ, der eine Schnittstelle ist.</span><span class="sxs-lookup"><span data-stu-id="fc199-125">For a type that is an interface.</span></span>|

<span data-ttu-id="fc199-129">Sie können mit der `--sig` -Compileroption eine Signaturdatei generieren.</span><span class="sxs-lookup"><span data-stu-id="fc199-129">You can generate a signature file by using the `--sig` compiler option.</span></span> <span data-ttu-id="fc199-130">Im Allgemeinen werden FSI-Dateien nicht manuell geschrieben.</span><span class="sxs-lookup"><span data-stu-id="fc199-130">Generally, you do not write .fsi files manually.</span></span> <span data-ttu-id="fc199-131">Stattdessen generieren Sie FSI-Dateien mit dem Compiler, fügen sie ggf. dem Projekt hinzu und bearbeiten sie, indem Sie Methoden und Funktionen entfernen, auf die kein Zugriff möglich sein soll.</span><span class="sxs-lookup"><span data-stu-id="fc199-131">Instead, you generate .fsi files by using the compiler, add them to your project, if you have one, and edit them by removing methods and functions that you do not want to be accessible.</span></span>

<span data-ttu-id="fc199-132">Für Typsignaturen gelten mehrere Regeln:</span><span class="sxs-lookup"><span data-stu-id="fc199-132">There are several rules for type signatures:</span></span>

- <span data-ttu-id="fc199-134">Datensätze und Unterscheidungs-Unions müssen entweder alle oder keine der zugehörigen Felder und Konstruktoren verfügbar machen, und die Reihenfolge in der Signatur muss mit der Reihenfolge in der Implementierungsdatei übereinstimmen.</span><span class="sxs-lookup"><span data-stu-id="fc199-134">Records and discriminated unions must expose either all or none of their fields and constructors, and the order in the signature must match the order in the implementation file.</span></span> <span data-ttu-id="fc199-135">Klassen können einige, alle oder keine ihrer Felder und Methoden in der Signatur offenlegen.</span><span class="sxs-lookup"><span data-stu-id="fc199-135">Classes can reveal some, all, or none of their fields and methods in the signature.</span></span>
