# <a name="native-wpf-browser-hosting-support-apis"></a>Systemeigene WPF-APIs mit Unterstützung für das Hosten in Browsern
Das Hosten von WPF-Anwendungen in Webbrowsern wird durch einen Active Document-Server (auch als DocObject bezeichnet) erleichtert, der vom WPF-Host registriert ist. Internet Explorer kann direkt aktiviert und in ein aktives Dokument integriert werden. Für das Hosten von XBAPs und lose XAML-Dokumenten in Mozilla-Browsern stellt WPF ein NPAPI-Plugin bereit, das eine ähnliche Hosting-Umgebung wie der WPF Active Document-Server bereitstellt wie Internet Explorer. Die einfachste praktische Möglichkeit zum Hosten von XBAPs und XAML-Dokumenten in anderen Browsern und eigenständigen Anwendungen ist jedoch über das Internet Explorer-Webbrowsersteuerelement. Das Webbrowsersteuerelement stellt die komplexe Active Document-Serverhostingumgebung bereit, ermöglicht jedoch seinem eigenen Host das Anpassen und Erweitern dieser Umgebung sowie die direkte Kommunikation mit dem aktuellen Active Document-Objekt.  
  
 Der WPF Active Document-Server implementiert mehrere gängige Hostingschnittstellen, einschließlich [IOleObject](/windows/win32/api/oleidl/nn-oleidl-ioleobject), [IOleDocument](/windows/win32/api/docobj/nn-docobj-ioledocument), [IOleInPlaceActiveObject](/windows/win32/api/oleidl/nn-oleidl-ioleinplaceactiveobject), [IPersistMoniker](https://docs.microsoft.com/previous-versions/windows/internet-explorer/ie-developer/platform-apis/ms775042(v=vs.85)), [IOleCommandTarget](/windows/win32/api/docobj/nn-docobj-iolecommandtarget). Beim Gehostet im Webbrowser-Steuerelement können diese Schnittstellen Abfragen aus dem Objekt sein, das von der [IWebBrowser2::Document-Eigenschaft](https://docs.microsoft.com/previous-versions/aa752116(v=vs.85)) zurückgegeben wird.  
  
## <a name="iolecommandtarget"></a>Iolecommandtarget  
 Die Implementierung von [IOleCommandTarget](/windows/win32/api/docobj/nn-docobj-iolecommandtarget) durch den WPF Active Document-Server unterstützt zahlreiche navigationsbezogene und browserspezifische Befehle der Ole-Standardbefehlsgruppe (mit einer NULL-Befehlsgruppe GUID). Darüber hinaus erkennt es eine benutzerdefinierte Befehlsgruppe namens CGID_PresentationHost. Derzeit ist nur ein Befehl innerhalb dieser Gruppe definiert.  
  
```cpp  
DEFINE_GUID(CGID_PresentationHost, 0xd0288c55, 0xd6, 0x4f5e, 0xa8, 0x51, 0x79, 0xde, 0xc5, 0x1b, 0x10, 0xec);  
enum PresentationHostCommands {
   PHCMDID_TABINTO = 1
};  
```  
  
 PHCMDID_TABINTO weist PresentationHost an, den Fokus auf das erste oder letzte fokussierbare Element in seinem Inhalt zu wechseln, abhängig vom Status der Umschalttaste.
