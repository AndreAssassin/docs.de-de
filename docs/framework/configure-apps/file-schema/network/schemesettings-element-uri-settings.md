## <a name="remarks"></a><span data-ttu-id="1086a-126">Bemerkungen</span><span class="sxs-lookup"><span data-stu-id="1086a-126">Remarks</span></span>  
 <span data-ttu-id="1086a-127">Standardmäßig entweicht <xref:System.Uri?displayProperty=nameWithType> die Klasse prozentweise codierte Pfadtrennzeichen, bevor die Pfadkomprimierung ausgeführt wird.</span><span class="sxs-lookup"><span data-stu-id="1086a-127">By default, the <xref:System.Uri?displayProperty=nameWithType> class un-escapes percent encoded path delimiters before executing path compression.</span></span> <span data-ttu-id="1086a-128">Dies wurde als Sicherheitsmechanismus gegen Angriffe wie die folgenden implementiert:</span><span class="sxs-lookup"><span data-stu-id="1086a-128">This was implemented as a security mechanism against attacks like the following:</span></span>  
  
 `http://www.contoso.com/..%2F..%2F/Windows/System32/cmd.exe?/c+dir+c:\`  
  
 <span data-ttu-id="1086a-129">Wenn dieser URI an Module weitergegeben wird, die prozentisch codierte Zeichen nicht korrekt verarbeiten, kann dies dazu führen, dass der folgende Befehl vom Server ausgeführt wird:</span><span class="sxs-lookup"><span data-stu-id="1086a-129">If this URI gets passed down to modules not handling percent encoded characters correctly, it could result in the following command being executed by the server:</span></span>  
  
 `c:\Windows\System32\cmd.exe /c dir c:\`  
  
 <span data-ttu-id="1086a-130">Aus diesem <xref:System.Uri?displayProperty=nameWithType> Grund werden Pfadtrennzeichen der Klasse zuerst nicht entweichen und dann die Pfadkomprimierung angewendet.</span><span class="sxs-lookup"><span data-stu-id="1086a-130">For this reason, <xref:System.Uri?displayProperty=nameWithType> class first un-escapes path delimiters and then applies path compression.</span></span> <span data-ttu-id="1086a-131">Das Ergebnis der Übergabe der <xref:System.Uri?displayProperty=nameWithType> schädlichen URL oben an den Klassenkonstruktor führt zu folgendem URI:</span><span class="sxs-lookup"><span data-stu-id="1086a-131">The result of passing the malicious URL above to <xref:System.Uri?displayProperty=nameWithType> class constructor results in the following URI:</span></span>  
  
 `http://www.microsoft.com/Windows/System32/cmd.exe?/c+dir+c:\`  
  
 <span data-ttu-id="1086a-132">Dieses Standardverhalten kann mithilfe der Konfigurationsoption schemeSettings für ein bestimmtes Schema geändert werden, um nicht das Escape-Prozent-codierte Pfadtrennzeichen zu entführen.</span><span class="sxs-lookup"><span data-stu-id="1086a-132">This default behavior can be modified to not un-escape percent encoded path delimiters using the schemeSettings configuration option for a specific scheme.</span></span>

## <a name="example"></a><span data-ttu-id="1086a-135">Beispiel</span><span class="sxs-lookup"><span data-stu-id="1086a-135">Example</span></span>  
 <span data-ttu-id="1086a-136">Das folgende Beispiel zeigt eine <xref:System.Uri> Konfiguration, die von der Klasse verwendet wird, um nicht aus percentcodierte Pfadtrennzeichen für das http-Schema zu entkommen.</span><span class="sxs-lookup"><span data-stu-id="1086a-136">The following example shows a configuration used by the <xref:System.Uri> class to support not escaping percent-encoded path delimiters for the http scheme.</span></span>
