## <a name="text-representation"></a><span data-ttu-id="5a3af-107">Textdarstellung</span><span class="sxs-lookup"><span data-stu-id="5a3af-107">Text Representation</span></span>

<span data-ttu-id="5a3af-108">Die folgenden drei konventionellen Formate werden verwendet, um die IPv6-Adressen als Textzeichenfolgen darzustellen:</span><span class="sxs-lookup"><span data-stu-id="5a3af-108">The following are the three conventional forms used to represent the IPv6 addresses as text strings:</span></span>

- <span data-ttu-id="5a3af-109">**Hexadezimalformat mit Doppelpunkt**.</span><span class="sxs-lookup"><span data-stu-id="5a3af-109">**Colon-hexadecimal form**.</span></span> <span data-ttu-id="5a3af-110">Das bevorzugte Format ist „n:n:n:n:n:n:n:n“.</span><span class="sxs-lookup"><span data-stu-id="5a3af-110">This is the preferred form n:n:n:n:n:n:n:n.</span></span> <span data-ttu-id="5a3af-111">Jedes „n“ stellt den hexadezimalen Wert von einem der acht 16-Bit-Elemente der Adresse dar.</span><span class="sxs-lookup"><span data-stu-id="5a3af-111">Each n represents the hexadecimal value of one of the eight 16-bit elements of the address.</span></span> <span data-ttu-id="5a3af-112">Beispiel: `3FFE:FFFF:7654:FEDA:1245:BA98:3210:4562`.</span><span class="sxs-lookup"><span data-stu-id="5a3af-112">For example: `3FFE:FFFF:7654:FEDA:1245:BA98:3210:4562`.</span></span>

- <span data-ttu-id="5a3af-113">**Komprimiertes Format**.</span><span class="sxs-lookup"><span data-stu-id="5a3af-113">**Compressed form**.</span></span> <span data-ttu-id="5a3af-114">Aufgrund der Adresslänge ist es üblich, dass die Adressen lange Zeichenfolgen enthalten, die aus Nullen (0) bestehen.</span><span class="sxs-lookup"><span data-stu-id="5a3af-114">Due to the address length, it is common to have addresses containing a long string of zeros.</span></span> <span data-ttu-id="5a3af-115">Verwenden Sie das komprimierte Format, um das Schreiben dieser Adressen zu vereinfachen. In diesem wird eine einzelne zusammenhängende Folge von 0-Blöcken durch zwei Doppelpunkte (::) dargestellt.</span><span class="sxs-lookup"><span data-stu-id="5a3af-115">To simplify writing these addresses, use the compressed form, in which a single contiguous sequence of 0 blocks are represented by a double-colon symbol (::).</span></span> <span data-ttu-id="5a3af-116">Dieses Symbol kann in einer Adresse nur einmal vorkommen.</span><span class="sxs-lookup"><span data-stu-id="5a3af-116">This symbol can appear only once in an address.</span></span> <span data-ttu-id="5a3af-117">Die Multicastadresse `FFED:0:0:0:0:BA98:3210:4562` ist in komprimierter Form zum Beispiel `FFED::BA98:3210:4562`.</span><span class="sxs-lookup"><span data-stu-id="5a3af-117">For example, the multicast address `FFED:0:0:0:0:BA98:3210:4562` in compressed form is `FFED::BA98:3210:4562`.</span></span> <span data-ttu-id="5a3af-118">Die Unicastadresse `3FFE:FFFF:0:0:8:800:20C4:0` ist in komprimierter Form `3FFE:FFFF::8:800:20C4:0`.</span><span class="sxs-lookup"><span data-stu-id="5a3af-118">The unicast address `3FFE:FFFF:0:0:8:800:20C4:0` in compressed form is `3FFE:FFFF::8:800:20C4:0`.</span></span> <span data-ttu-id="5a3af-119">Die Loopbackadresse `0:0:0:0:0:0:0:1` ist in komprimierter Form `::`1.</span><span class="sxs-lookup"><span data-stu-id="5a3af-119">The loopback address `0:0:0:0:0:0:0:1` in compressed form is `::`1.</span></span> <span data-ttu-id="5a3af-120">Die nicht definierte Adresse `0:0:0:0:0:0:0:0` ist in komprimierter Form `::`.</span><span class="sxs-lookup"><span data-stu-id="5a3af-120">The unspecified address `0:0:0:0:0:0:0:0` in compressed form is `::`.</span></span>

- <span data-ttu-id="5a3af-121">**Mixed form (Gemischtes Format)** .</span><span class="sxs-lookup"><span data-stu-id="5a3af-121">**Mixed form**.</span></span> <span data-ttu-id="5a3af-122">Dieses Format kombiniert IPv4- und IPv6-Adressen.</span><span class="sxs-lookup"><span data-stu-id="5a3af-122">This form combines IPv4 and IPv6 addresses.</span></span> <span data-ttu-id="5a3af-123">In diesem Fall ist das Adressformat „n:n:n:n:n:n:d.d.d.d“, bei dem jedes „n“ den Hexadezimalwert der 6 oberen Elemente der IPv6-16-Bit-Adresse darstellt und jedes „d“ den Dezimalwert der IPv4-Adresse.</span><span class="sxs-lookup"><span data-stu-id="5a3af-123">In this case, the address format is n:n:n:n:n:n:d.d.d.d, where each n represents the hexadecimal values of the six IPv6 high-order 16-bit address elements, and each d represents the decimal value of an IPv4 address.</span></span>

## <a name="address-types"></a><span data-ttu-id="5a3af-124">Adresstypen</span><span class="sxs-lookup"><span data-stu-id="5a3af-124">Address Types</span></span>

<span data-ttu-id="5a3af-127">Eine IPv6-Unicastadresse ist in zwei Teile unterteilt.</span><span class="sxs-lookup"><span data-stu-id="5a3af-127">An IPv6 unicast address is divided into two parts.</span></span> <span data-ttu-id="5a3af-128">Der erste Teil enthält das Adresspräfix, und der zweite Teil enthält den Schnittstellenbezeichner.</span><span class="sxs-lookup"><span data-stu-id="5a3af-128">The first part contains the address prefix, and the second part contains the interface identifier.</span></span> <span data-ttu-id="5a3af-129">Bei Folgendem handelt es sich um eine präzise Methode, um die Kombination aus einer IPv6-Adresse und einem Präfix auszudrücken: IPv6-Adresse/Präfixlänge.</span><span class="sxs-lookup"><span data-stu-id="5a3af-129">A concise way to express an IPv6 address/prefix combination is as follows: ipv6-address/prefix-length.</span></span>

<span data-ttu-id="5a3af-130">Das folgende Beispiel enthält eine Adresse mit einem 64-Bit-Präfix.</span><span class="sxs-lookup"><span data-stu-id="5a3af-130">The following is an example of an address with a 64-bit prefix.</span></span>

<span data-ttu-id="5a3af-131">`3FFE:FFFF:0:CD30:0:0:0:0/64`.</span><span class="sxs-lookup"><span data-stu-id="5a3af-131">`3FFE:FFFF:0:CD30:0:0:0:0/64`.</span></span>

<span data-ttu-id="5a3af-132">Das Präfix in diesem Beispiel ist `3FFE:FFFF:0:CD30`.</span><span class="sxs-lookup"><span data-stu-id="5a3af-132">The prefix in this example is `3FFE:FFFF:0:CD30`.</span></span> <span data-ttu-id="5a3af-133">Die Adresse kann auch in einem komprimierten Format (`3FFE:FFFF:0:CD30::/64`) geschrieben werden.</span><span class="sxs-lookup"><span data-stu-id="5a3af-133">The address can also be written in a compressed form, as `3FFE:FFFF:0:CD30::/64`.</span></span>

<span data-ttu-id="5a3af-134">IPv6 definiert die folgenden Adresstypen:</span><span class="sxs-lookup"><span data-stu-id="5a3af-134">IPv6 defines the following address types:</span></span>

- <span data-ttu-id="5a3af-135">**Unicast address (Unicastadressen)** .</span><span class="sxs-lookup"><span data-stu-id="5a3af-135">**Unicast address**.</span></span> <span data-ttu-id="5a3af-136">Ein Bezeichner für eine einzelne Schnittstelle.</span><span class="sxs-lookup"><span data-stu-id="5a3af-136">An identifier for a single interface.</span></span> <span data-ttu-id="5a3af-137">Ein Paket, das an diese Adresse gesendet wird, wird an die angegebene Schnittstelle übermittelt.</span><span class="sxs-lookup"><span data-stu-id="5a3af-137">A packet sent to this address is delivered to the identified interface.</span></span> <span data-ttu-id="5a3af-138">Die Unicastadressen werden von den Multicastadressen durch den Wert des oberen Oktetts unterschieden.</span><span class="sxs-lookup"><span data-stu-id="5a3af-138">The unicast addresses are distinguished from the multicast addresses by the value of the high-order octet.</span></span> <span data-ttu-id="5a3af-139">Das obere Oktett der Multicastadressen hat den Hexadezimalwert von FF.</span><span class="sxs-lookup"><span data-stu-id="5a3af-139">The multicast addresses' high-order octet has the hexadecimal value of FF.</span></span> <span data-ttu-id="5a3af-140">Jeder andere Wert für dieses Oktett bezeichnet eine Unicastadresse.</span><span class="sxs-lookup"><span data-stu-id="5a3af-140">Any other value for this octet identifies a unicast address.</span></span> <span data-ttu-id="5a3af-141">Es gibt folgende verschiedene Typen von Unicastadressen:</span><span class="sxs-lookup"><span data-stu-id="5a3af-141">The following are different types of unicast addresses:</span></span>

  - <span data-ttu-id="5a3af-142">**Link-local addresses (Verbindungslokale Adressen)** .</span><span class="sxs-lookup"><span data-stu-id="5a3af-142">**Link-local addresses**.</span></span> <span data-ttu-id="5a3af-143">Diese Adressen werden dann in einem einzigen Link verwendet und weisen folgendes Format auf: FE80::*InterfaceID*.</span><span class="sxs-lookup"><span data-stu-id="5a3af-143">These addresses are used on a single link and have the following format: FE80::*InterfaceID*.</span></span> <span data-ttu-id="5a3af-144">Verbindungslokale Adressen werden zwischen Knoten auf einem Link zur automatischen Adresskonfiguration und Nachbarsuche verwendet oder wenn keine Router vorhanden sind.</span><span class="sxs-lookup"><span data-stu-id="5a3af-144">Link-local addresses are used between nodes on a link for auto-address configuration, neighbor discovery, or when no routers are present.</span></span> <span data-ttu-id="5a3af-145">Eine verbindungslokale Adresse wird in erster Linie beim Start verwendet und wenn das System noch keine größeren Adressen abgerufen hat.</span><span class="sxs-lookup"><span data-stu-id="5a3af-145">A link-local address is used primarily at startup and when the system has not yet acquired addresses of larger scope.</span></span>

- <span data-ttu-id="5a3af-151">**Multicast address (Multicastadressen)** .</span><span class="sxs-lookup"><span data-stu-id="5a3af-151">**Multicast address**.</span></span> <span data-ttu-id="5a3af-152">Ein Bezeichner für eine Reihe von Schnittstellen, die normalerweise zu den unterschiedlichen Knoten gehören.</span><span class="sxs-lookup"><span data-stu-id="5a3af-152">An identifier for a set of interfaces (typically belonging to different nodes).</span></span> <span data-ttu-id="5a3af-153">Ein Paket, das an diese Adresse gesendet wird, wird an alle von der Adresse angegebenen Schnittstellen gesendet.</span><span class="sxs-lookup"><span data-stu-id="5a3af-153">A packet sent to this address is delivered to all the interfaces identified by the address.</span></span> <span data-ttu-id="5a3af-154">Die Multicastadresstypen ersetzen die IPv4-Broadcastadressen.</span><span class="sxs-lookup"><span data-stu-id="5a3af-154">The multicast address types supersede the IPv4 broadcast addresses.</span></span>

- <span data-ttu-id="5a3af-155">**Anycast address (Anycastadressen)** .</span><span class="sxs-lookup"><span data-stu-id="5a3af-155">**Anycast address**.</span></span> <span data-ttu-id="5a3af-156">Ein Bezeichner für eine Reihe von Schnittstellen, die normalerweise zu den unterschiedlichen Knoten gehören.</span><span class="sxs-lookup"><span data-stu-id="5a3af-156">An identifier for a set of interfaces (typically belonging to different nodes).</span></span> <span data-ttu-id="5a3af-157">Ein Paket, das an diese Adresse gesendet wird, wird nur an eine von der Adresse angegebenen Schnittstellen gesendet.</span><span class="sxs-lookup"><span data-stu-id="5a3af-157">A packet sent to this address is delivered to only one interface identified by the address.</span></span> <span data-ttu-id="5a3af-158">Dabei handelt es sich um die Schnittstelle, die von den Routingmetriken als die nächstgelegene angegeben wird.</span><span class="sxs-lookup"><span data-stu-id="5a3af-158">This is the nearest interface as identified by routing metrics.</span></span> <span data-ttu-id="5a3af-159">Anycastadressen werden aus dem Unicastadressraum erstellt und sind syntaktisch nicht unterscheidbar.</span><span class="sxs-lookup"><span data-stu-id="5a3af-159">Anycast addresses are taken from the unicast address space and are not syntactically distinguishable.</span></span> <span data-ttu-id="5a3af-160">Die adressierte Schnittstelle nimmt die Unterscheidung zwischen Unicast- und Anycastadressen als Funktion ihrer Konfiguration vor.</span><span class="sxs-lookup"><span data-stu-id="5a3af-160">The addressed interface performs the distinction between unicast and anycast addresses as a function of its configuration.</span></span>
