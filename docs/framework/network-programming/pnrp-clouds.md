# <a name="pnrp-clouds"></a><span data-ttu-id="8330e-102">PNRP-Clouds</span><span class="sxs-lookup"><span data-stu-id="8330e-102">PNRP Clouds</span></span>

 <span data-ttu-id="8330e-105">Die Kommunikation zwischen den Knoten sollte die Grenze einer Cloud nie überschreiten.</span><span class="sxs-lookup"><span data-stu-id="8330e-105">Communication between nodes should never cross from one cloud to another.</span></span> <span data-ttu-id="8330e-106">Eine <xref:System.Net.PeerToPeer.Cloud>-Instanz wird durch ihren Namen eindeutig bestimmt. Dabei muss Groß- und Kleinschreibung beachtet werden.</span><span class="sxs-lookup"><span data-stu-id="8330e-106">A <xref:System.Net.PeerToPeer.Cloud> instance is uniquely identified by its name, which is case-sensitive.</span></span> <span data-ttu-id="8330e-107">Ein einzelner Peer oder Knoten kann mit mehr als einer Cloud verbunden sein.</span><span class="sxs-lookup"><span data-stu-id="8330e-107">A single peer or node may be connected to more than one cloud.</span></span>  
  
 <span data-ttu-id="8330e-108">Clouds sind sehr eng an Netzwerkschnittstellen gebunden.</span><span class="sxs-lookup"><span data-stu-id="8330e-108">Clouds are tied very closely to network interfaces.</span></span>  <span data-ttu-id="8330e-109">Auf einem mehrfach vernetzten Computer mit zwei Netzwerkkarten, die mit unterschiedlichen Subnetzen verbunden sind, werden drei Clouds zurückgegeben: eine für jede lokale Linkadresse pro Schnittstelle und eine einzelne Cloud mit globalem Gültigkeitsbereich.</span><span class="sxs-lookup"><span data-stu-id="8330e-109">On a multi-homed machine with two network cards attached to different subnets, three clouds will be returned: one for each of the link local addresses per interface, and a single global scope cloud.</span></span>  
  
 <span data-ttu-id="8330e-110">PNRP verwendet drei Cloudbereiche, wobei diese Bereiche eine Gruppe von Computern darstellen, die sich gegenseitig finden können:</span><span class="sxs-lookup"><span data-stu-id="8330e-110">PNRP uses three cloud "scopes", in which a scope is a grouping of computers that are able to find each other:</span></span>  
  
-   <span data-ttu-id="8330e-111">Die globale Cloud entspricht dem globalen IPv6-Adressbereich sowie globalen Adressen. Sie stellt alle Computer des gesamten IPv6-Internets dar.</span><span class="sxs-lookup"><span data-stu-id="8330e-111">The global cloud corresponds to the global IPv6 address scope and global addresses and represents all the computers on the entire IPv6 Internet.</span></span> <span data-ttu-id="8330e-112">Es gibt nur eine einzige globale Cloud.</span><span class="sxs-lookup"><span data-stu-id="8330e-112">There is only a single global cloud.</span></span>  
  
-   <span data-ttu-id="8330e-113">Die verbindungslokale Cloud entspricht dem verbindungslokalen IPv6-Adressbereich sowie verbindungslokalen Adressen.</span><span class="sxs-lookup"><span data-stu-id="8330e-113">The link-local cloud corresponds to the link-local IPv6 address scope and link-local addresses.</span></span> <span data-ttu-id="8330e-114">Eine verbindungslokale Cloud gilt für einen bestimmten Link, der in der Regel mit dem lokal verbundenen Subnetz identisch ist.</span><span class="sxs-lookup"><span data-stu-id="8330e-114">A link-local cloud is for a specific link, which is typically the same as the locally attached subnet.</span></span> <span data-ttu-id="8330e-115">Es können mehrere verbindungslokale Clouds vorhanden sein.</span><span class="sxs-lookup"><span data-stu-id="8330e-115">There can be multiple link-local clouds.</span></span>  
  
 <span data-ttu-id="8330e-116">Eine dritte Möglichkeit ist die standortspezifische Cloud, die dem standortspezifischen IPv6-Adressbereich sowie standortlokalen Adressen entspricht.</span><span class="sxs-lookup"><span data-stu-id="8330e-116">A third cloud, the site-specific cloud, corresponds to the site IPv6 address scope and site-local addresses.</span></span> <span data-ttu-id="8330e-117">Diese Cloud ist veraltet, wird jedoch weiterhin in PNRP unterstützt.</span><span class="sxs-lookup"><span data-stu-id="8330e-117">This cloud has been deprecated, although it is still supported in PNRP.</span></span>  
  
## <a name="clouds"></a><span data-ttu-id="8330e-118">Clouds</span><span class="sxs-lookup"><span data-stu-id="8330e-118">Clouds</span></span>  
 <span data-ttu-id="8330e-119">PNRP-Clouds werden durch Instanzen der <xref:System.Net.PeerToPeer.Cloud>-Klasse dargestellt.</span><span class="sxs-lookup"><span data-stu-id="8330e-119">PNRP clouds are represented by instances of the <xref:System.Net.PeerToPeer.Cloud> class.</span></span> <span data-ttu-id="8330e-120">Gruppen von Clouds, die einen Peer verwenden, werden durch Instanzen der Enumerable-Klasse <xref:System.Net.PeerToPeer.CloudCollection> dargestellt.</span><span class="sxs-lookup"><span data-stu-id="8330e-120">Groups of clouds used a peer are represented by instances of the enumerable <xref:System.Net.PeerToPeer.CloudCollection> class.</span></span> <span data-ttu-id="8330e-121">Sammlungen von PNRP-Clouds, die dem aktuellen Peer bekannt sind, können durch Aufrufen der statischen <xref:System.Net.PeerToPeer.Cloud.GetAvailableClouds%2A>-Methode abgerufen werden.</span><span class="sxs-lookup"><span data-stu-id="8330e-121">Collections of PNRP clouds known to the current peer can be obtained by calling the static <xref:System.Net.PeerToPeer.Cloud.GetAvailableClouds%2A> method.</span></span>
