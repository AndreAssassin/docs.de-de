# <a name="peer-names-and-pnrp-ids"></a><span data-ttu-id="b1121-102">Peernamen und PNRP-IDs</span><span class="sxs-lookup"><span data-stu-id="b1121-102">Peer Names and PNRP IDs</span></span>
<span data-ttu-id="b1121-103">Ein Peername steht für einen Endpunkt für die Kommunikation, der ein Computer, ein Benutzer, eine Gruppe, ein Dienst oder alles sein kein, was einem Peer zugeordnet ist, das in eine IPv6-Adresse aufgelöst werden kann.</span><span class="sxs-lookup"><span data-stu-id="b1121-103">A Peer Name represents an endpoint for communication, which can be a computer, a user, a group, a service, or anything associated with a Peer that can be resolved to an IPv6 address.</span></span> <span data-ttu-id="b1121-104">Das Peer Name Resolution-Protokoll (PNRP) nimmt den statistisch eindeutigen Peernamen zur Erstellung einer PNRP-ID an, die zur Identifikation von Cloudelementen verwendet wird.</span><span class="sxs-lookup"><span data-stu-id="b1121-104">The Peer Name Resolution Protocol (PNRP) takes the statistically unique Peer Name for the creation of a PNRP ID, which is used to identify cloud members.</span></span>  
  
## <a name="peer-names"></a><span data-ttu-id="b1121-105">Peernamen</span><span class="sxs-lookup"><span data-stu-id="b1121-105">Peer Names</span></span>  
 <span data-ttu-id="b1121-106">Peernamen können unsicher oder gesichert gespeichert werden.</span><span class="sxs-lookup"><span data-stu-id="b1121-106">Peer names can be registered as unsecured or secured.</span></span> <span data-ttu-id="b1121-107">Unsichere Namen sind nur Textzeichenfolgen, die den Gefahren von Spoofing ausgesetzt sind, da jeder einen ungesicherten doppelten Namen registrieren kann.</span><span class="sxs-lookup"><span data-stu-id="b1121-107">Unsecured names are just text strings that are subject to spoofing, as anyone can register a duplicate unsecured name.</span></span> <span data-ttu-id="b1121-108">Unsichere Namen werden am besten in privaten oder anderweitig geschützten Netzwerken verwendet.</span><span class="sxs-lookup"><span data-stu-id="b1121-108">Unsecured names are best used in private or otherwise protected networks.</span></span> <span data-ttu-id="b1121-109">Gesicherte Namen werden durch ein Zertifikat und eine digitale Signatur geschützt.</span><span class="sxs-lookup"><span data-stu-id="b1121-109">Secured names are protected with a certificate and a digital signature.</span></span> <span data-ttu-id="b1121-110">Nur der ursprüngliche Verleger kann den Besitz eines sicheren Namens nachweisen.</span><span class="sxs-lookup"><span data-stu-id="b1121-110">Only the original publisher will be able to prove ownership of a secured name.</span></span>  
  
 <span data-ttu-id="b1121-111">Die Kombination von Cloud und Umfang bietet eine angemessen sichere Umgebung für Peers, die an der PNRP-Aktivität beteiligt sind.</span><span class="sxs-lookup"><span data-stu-id="b1121-111">The combination of cloud and scope provides a reasonably secure environment for peers that participate in PNRP activity.</span></span> <span data-ttu-id="b1121-112">Die Verwendung eines sicheren Peernamens garantiert jedoch nicht die Gesamtsicherheit der Netzwerkanwendung.</span><span class="sxs-lookup"><span data-stu-id="b1121-112">However, using a secured peer name does not ensure the overall security of the networking application.</span></span> <span data-ttu-id="b1121-113">Die Sicherheit der Anwendung ist implementierungsabhängig.</span><span class="sxs-lookup"><span data-stu-id="b1121-113">Security of the application is implementation-dependent.</span></span>  
  
 <span data-ttu-id="b1121-114">Gesicherte Peernamen werden nur vom Besitzer registriert und mit einem öffentlichen kryptografischen Schlüssel geschützt.</span><span class="sxs-lookup"><span data-stu-id="b1121-114">Secured peer names are only registered by their owner and are protected with public key cryptography.</span></span> <span data-ttu-id="b1121-115">Ein gesicherter Peername ist im Besitz der Peerentität, die über den entsprechenden privaten Schlüssel verfügt.</span><span class="sxs-lookup"><span data-stu-id="b1121-115">A secured peer name is considered owned by the peer entity having the corresponding private key.</span></span> <span data-ttu-id="b1121-116">Über die zertifizierte Peeradresse (CPA), die mit dem privaten Schlüssel signiert ist, kann der Besitz nachgewiesen werden.</span><span class="sxs-lookup"><span data-stu-id="b1121-116">Ownership can be proved via the certified peer address (CPA), which is signed using the private key.</span></span> <span data-ttu-id="b1121-117">Ein böswilliger Benutzer kann den Besitz eines Peernamen ohne den entsprechenden privaten Schlüssel nicht erstellen.</span><span class="sxs-lookup"><span data-stu-id="b1121-117">A malicious user cannot forge ownership of a peer name without the corresponding private key.</span></span>  
  
## <a name="pnrp-ids"></a><span data-ttu-id="b1121-118">PNRP-IDs</span><span class="sxs-lookup"><span data-stu-id="b1121-118">PNRP IDs</span></span>  
 <span data-ttu-id="b1121-119">![PNRP ID](./media/fdc9e8a0-4a1c-488d-a019-bc3a1973220c.gif "fdc9e8a0-4a1c-488d-a019-bc3a1973220c")</span><span class="sxs-lookup"><span data-stu-id="b1121-119">![PNRP ID](./media/fdc9e8a0-4a1c-488d-a019-bc3a1973220c.gif "fdc9e8a0-4a1c-488d-a019-bc3a1973220c")</span></span>  
  
 <span data-ttu-id="b1121-120">PNRP-IDs bestehen aus den folgenden Teilen:</span><span class="sxs-lookup"><span data-stu-id="b1121-120">PNRP IDs are composed of the following:</span></span>  
  
- <span data-ttu-id="b1121-121">Die höherwertigen 128 Bits sind als Peer-to-Peer-ID (P2P) bekannt. Sie sind ein Hash des Peernamens, der dem Endpunkt zugewiesen ist.</span><span class="sxs-lookup"><span data-stu-id="b1121-121">The high-order 128 bits, known as the peer-to-peer (P2P) ID, are a hash of a peer name assigned to the endpoint.</span></span> <span data-ttu-id="b1121-122">Der Peername hat das folgende Format: *Authority.Classifier*.</span><span class="sxs-lookup"><span data-stu-id="b1121-122">The peer name has the following format: *Authority.Classifier*.</span></span> <span data-ttu-id="b1121-123">Für sichere Namen ist *Autorität* der Secure-Hash-Algorithmus 1 (SHA1) des öffentlichen Schlüssels des Peernamens in hexadezimalen Zeichen.</span><span class="sxs-lookup"><span data-stu-id="b1121-123">For secured names, *Authority* is the Secure Hash Algorithm 1 (SHA1) hash of the public key of the peer name in hexadecimal characters.</span></span> <span data-ttu-id="b1121-124">Für unsichere Namen ist *Autorität* als einzelnes Zeichen „0“ festgelegt.</span><span class="sxs-lookup"><span data-stu-id="b1121-124">For unsecured names, the *Authority* is the single character "0".</span></span> <span data-ttu-id="b1121-125">*Klassifizierung* ist eine Zeichenfolge, die die Anwendung identifiziert.</span><span class="sxs-lookup"><span data-stu-id="b1121-125">*Classifier* is a string that identifies the application.</span></span> <span data-ttu-id="b1121-126">Kein Klassifizierer für Peernamen kann länger als 149 Zeichen sein, einschließlich dem `null`-Abschlusszeichen.</span><span class="sxs-lookup"><span data-stu-id="b1121-126">No peer name classifier can be greater than 149 characters long, including the `null` terminator.</span></span>  
  
- <span data-ttu-id="b1121-127">Die niederwertigen 128 Bits werden für die Dienstidentifizierung verwendet. Dies ist eine generierte Zahl, die verschiedene Instanzen der gleichen P2P-ID in der gleichen Cloud identifiziert.</span><span class="sxs-lookup"><span data-stu-id="b1121-127">The low-order 128 bits are used for the Service Location, which is a generated number that identifies different instances of the same P2P ID in the same cloud.</span></span>  
  
 <span data-ttu-id="b1121-128">Diese Kombination von P2P-ID und Dienstidentifizierung ermöglicht die Registrierung mehrerer PNRP-IDs von einem einzelnen Computer aus.</span><span class="sxs-lookup"><span data-stu-id="b1121-128">This combination of P2P ID and Service Location allows multiple PNRP IDs to be registered from a single computer.</span></span>
