# <a name="deprecated-clr-hosting-interfaces-and-coclasses"></a>Veraltete CLR-Hostingschnittstellen und Co-Klassen
In diesem Abschnitt wird beschrieben, die Schnittstellen, die nicht verwaltete Hosts können in den .NET Framework-Versionen 1.0 und 1.1 in ihre Anwendungen integrieren die common Language Runtime (CLR) verwenden. Diese Schnittstellen bieten Methoden, die für einen Host zu konfigurieren und die Runtime in einen Prozess geladen.  
  
## <a name="in-this-section"></a>In diesem Abschnitt  
 IAppDomainSetup  
 Stellt Methoden für den Host so konfigurieren Sie eine <xref:System.AppDomain>.  
  
 [ICeeFileGen-Klasse](../../../../docs/framework/unmanaged-api/hosting/iceefilegen-class.md)  
 (Veraltet) Stellt Funktionen zum Erstellen einer native portable ausführbare Datei (PE) Datei bereit.  
  
 [ICorRuntimeHost-Schnittstelle](../../../../docs/framework/unmanaged-api/hosting/icorruntimehost-interface.md)  
 Stellt Methoden für den Host aus, um CLR-Einstellungen zu konfigurieren.
