# <a name="replay-attacks"></a><span data-ttu-id="9bf2f-102">Wiederholungsangriffe</span><span class="sxs-lookup"><span data-stu-id="9bf2f-102">Replay Attacks</span></span>
<span data-ttu-id="9bf2f-103">Ein *Wiederholungsangriff* tritt auf, wenn ein Angreifer einen Nachrichtenstream zwischen zwei Parteien kopiert und den Strom für eine oder mehrere Parteien wiedergibt.</span><span class="sxs-lookup"><span data-stu-id="9bf2f-103">A *replay attack* occurs when an attacker copies a stream of messages between two parties and replays the stream to one or more of the parties.</span></span> <span data-ttu-id="9bf2f-104">Wenn der Angriff nicht abgeschwächt wird, verarbeiten die angegriffenen Computer den Stream wie zulässige Nachrichten, was eine Reihe negativer Konsequenzen wie redundante Bestellungen eines Artikels zur Folge hat.</span><span class="sxs-lookup"><span data-stu-id="9bf2f-104">Unless mitigated, the computers subject to the attack process the stream as legitimate messages, resulting in a range of bad consequences, such as redundant orders of an item.</span></span>  
  
## <a name="bindings-may-be-subject-to-reflection-attacks"></a><span data-ttu-id="9bf2f-105">Bindungen sind möglicherweise Reflektionsangriffen ausgesetzt</span><span class="sxs-lookup"><span data-stu-id="9bf2f-105">Bindings May Be Subject to Reflection Attacks</span></span>  
 <span data-ttu-id="9bf2f-106">*Reflektionsangriffe* Replays von Nachrichten an einen Sender sind, als ob sie vom Empfänger der Antwort stammt.</span><span class="sxs-lookup"><span data-stu-id="9bf2f-106">*Reflection attacks* are replays of messages back to a sender as if they came from the receiver as the reply.</span></span> <span data-ttu-id="9bf2f-107">Der Standard *replay-Erkennung* in der Windows Communication Foundation (WCF) Mechanismus wird nicht automatisch behandelt.</span><span class="sxs-lookup"><span data-stu-id="9bf2f-107">The standard *replay detection* in the Windows Communication Foundation (WCF) mechanism does not automatically handle this.</span></span>  
  
 <span data-ttu-id="9bf2f-108">Reflektionsangriffe werden standardmäßig abgeschwächt, da die WCF-Dienstmodells eine signierte Nachrichtenkennung, für Anforderungsnachrichten hinzufügt und eine signierte erwartet `relates-to` Header in Antwortnachrichten.</span><span class="sxs-lookup"><span data-stu-id="9bf2f-108">Reflection attacks are mitigated by default because the WCF service model adds a signed message ID to request messages and expects a signed `relates-to` header on response messages.</span></span> <span data-ttu-id="9bf2f-109">Infolgedessen kann die Anforderungsnachricht nicht als Antwort wiederholt werden.</span><span class="sxs-lookup"><span data-stu-id="9bf2f-109">Consequently, the request message cannot be replayed as a response.</span></span> <span data-ttu-id="9bf2f-110">In sicheren Szenarien mit zuverlässigem Messaging werden Reflektionsangriffe aus folgenden Gründen abgemildert:</span><span class="sxs-lookup"><span data-stu-id="9bf2f-110">In secure reliable message (RM) scenarios, reflection attacks are mitigated because:</span></span>  
  
-   <span data-ttu-id="9bf2f-111">Die Sequenzerstellung und die Sequenzerstellungsantwort verwenden unterschiedliche Nachrichtenschemas.</span><span class="sxs-lookup"><span data-stu-id="9bf2f-111">The create sequence and create sequence response message schemas are different.</span></span>  
  
-   <span data-ttu-id="9bf2f-112">Bei Simplexsequenzen können die vom Client gesendeten Nachrichten nicht wiedergegeben werden, da diese vom Client nicht interpretiert werden können.</span><span class="sxs-lookup"><span data-stu-id="9bf2f-112">For simplex sequences, sequence messages the client sends cannot be replayed back to it because the client cannot understand such messages.</span></span>  
  
-   <span data-ttu-id="9bf2f-113">Bei Duplexsequenzen müssen die beiden Sequenzkennungen eindeutig sein.</span><span class="sxs-lookup"><span data-stu-id="9bf2f-113">For duplex sequences, the two sequence IDs must be unique.</span></span> <span data-ttu-id="9bf2f-114">Ausgehende Nachrichten können daher nicht als eingehende Nachrichten wiederholt werden (alle Header und Texte werden ebenfalls signiert).</span><span class="sxs-lookup"><span data-stu-id="9bf2f-114">Thus, an outgoing sequence message cannot be replayed back as an incoming sequence message (all sequence headers and bodies are signed, too).</span></span>  
  
 <span data-ttu-id="9bf2f-115">Aus diesem Grund sind nur Bindungen ohne WS-Adressierung anfällig für Reflektionsangriffe: benutzerdefinierte Bindungen, deren WS-Adressierung deaktiviert ist und die mit symmetrischen Schlüsseln gesichert sind.</span><span class="sxs-lookup"><span data-stu-id="9bf2f-115">The only bindings that are susceptible to reflection attacks are those without WS-Addressing: custom bindings that have WS-Addressing disabled and use the symmetric key-based security.</span></span> <span data-ttu-id="9bf2f-116">Die <xref:System.ServiceModel.BasicHttpBinding> verwendet standardmäßig keine WS-Adressierung; die Absicherung durch symmetrische Schlüssel erfolgt jedoch auf eine Art und Weise, die sie anfällig gegenüber dieser Art von Angriffen macht.</span><span class="sxs-lookup"><span data-stu-id="9bf2f-116">The <xref:System.ServiceModel.BasicHttpBinding> does not use WS-Addressing by default, but it does not use symmetric key-based security in a way that allows it to be vulnerable to this attack.</span></span>  
  
 <span data-ttu-id="9bf2f-117">Zur Entschärfung für benutzerdefinierte Bindungen sollte auf einen Sicherheitskontext verzichtet werden, oder es sollten Header für die WS-Adressierung verwendet werden.</span><span class="sxs-lookup"><span data-stu-id="9bf2f-117">The mitigation for custom bindings is to not establish security context or to require WS-Addressing headers.</span></span>  
  
## <a name="web-farm-attacker-replays-request-to-multiple-nodes"></a><span data-ttu-id="9bf2f-118">Webfarm: Angreifer Wiederholen von Anforderungen an mehreren Knoten</span><span class="sxs-lookup"><span data-stu-id="9bf2f-118">Web Farm: Attacker Replays Request to Multiple Nodes</span></span>  
 <span data-ttu-id="9bf2f-119">Ein Client verwendet einen Dienst, der in einer Webfarm implementiert ist.</span><span class="sxs-lookup"><span data-stu-id="9bf2f-119">A client uses a service that is implemented on a Web farm.</span></span> <span data-ttu-id="9bf2f-120">Ein Angreifer wiederholt eine Anforderung, die an einen Knoten in der Farm gesendet wurde, für einen anderen Knoten in der Farm.</span><span class="sxs-lookup"><span data-stu-id="9bf2f-120">An attacker replays a request that was sent to one node in the farm to another node in the farm.</span></span> <span data-ttu-id="9bf2f-121">Zusätzlich wird beim Neustart eines Diensts der Wiederholungscache geleert, sodass der Angriff wiederholt werden kann.</span><span class="sxs-lookup"><span data-stu-id="9bf2f-121">In addition, if a service is restarted, the replay cache is flushed, allowing an attacker to replay the request.</span></span> <span data-ttu-id="9bf2f-122">(Der Cache enthält die bereits verwendeten Signaturen von Nachrichten und verhindert so, dass diese mehrfach verwendet werden können.</span><span class="sxs-lookup"><span data-stu-id="9bf2f-122">(The cache contains used, previously seen message signature values and prevents replays so those signatures can be used only once.</span></span> <span data-ttu-id="9bf2f-123">Wiederholungscaches werden nicht von mehreren Anwendungen in einer Webfarm verwendet.)</span><span class="sxs-lookup"><span data-stu-id="9bf2f-123">Replay caches are not shared across a Web farm.)</span></span>  
  
 <span data-ttu-id="9bf2f-124">Mögliche Entschärfungen:</span><span class="sxs-lookup"><span data-stu-id="9bf2f-124">Mitigations include:</span></span>  
  
-   <span data-ttu-id="9bf2f-125">Verwenden Sie die Nachrichtenmodussicherheit mit zustandsbehafteten Token für den Sicherheitskontext (mit oder ohne sichere Konversation).</span><span class="sxs-lookup"><span data-stu-id="9bf2f-125">Use message mode security with stateful security context tokens (with or without secure conversation enabled).</span></span> <span data-ttu-id="9bf2f-126">Weitere Informationen finden Sie unter [Vorgehensweise: Erstellen Sie einen Sicherheitskontext für eine sichere Sitzung Token](../../../../docs/framework/wcf/feature-details/how-to-create-a-security-context-token-for-a-secure-session.md).</span><span class="sxs-lookup"><span data-stu-id="9bf2f-126">For more information, see [How to: Create a Security Context Token for a Secure Session](../../../../docs/framework/wcf/feature-details/how-to-create-a-security-context-token-for-a-secure-session.md).</span></span>  
  
-   <span data-ttu-id="9bf2f-127">Konfigurieren Sie den Dienst für die Sicherheit auf Transportebene.</span><span class="sxs-lookup"><span data-stu-id="9bf2f-127">Configure the service to use transport-level security.</span></span>
