# <a name="adopting-windows-communication-foundation"></a><span data-ttu-id="bae74-102">Übernehmen von Windows Communication Foundation</span><span class="sxs-lookup"><span data-stu-id="bae74-102">Adopting Windows Communication Foundation</span></span>

<span data-ttu-id="bae74-103">Sie können auch Windows Communication Foundation (WCF) für neue Entwicklungen verwenden Sie beim Verwalten von vorhandener Anwendungen weiterhin über ASP.NET entwickelt wurden.</span><span class="sxs-lookup"><span data-stu-id="bae74-103">You can choose to use Windows Communication Foundation (WCF) for new development, while continuing to maintain existing applications developed using ASP.NET.</span></span> <span data-ttu-id="bae74-104">Da WCF die beste Wahl für die Erleichterung der Kommunikation mit Anwendungen, die mit .NET Framework in jedem Szenario erstellt werden soll, kann als ein Standardtool für eine Vielzahl von Software von Kommunikationsproblemen auf eine Weise zu lösen, die von ASP.NET dienen ist nicht möglich.</span><span class="sxs-lookup"><span data-stu-id="bae74-104">Because WCF is intended to be the most suitable choice for facilitating communication with applications built with the .NET Framework in any scenario, it can serve as a standard tool for solving a wide variety of software communications problems in a way that ASP.NET cannot.</span></span>

<span data-ttu-id="bae74-105">Neue WCF-Anwendungen können auf denselben Computern wie vorhandene ASP.NET-Webdienste bereitgestellt werden.</span><span class="sxs-lookup"><span data-stu-id="bae74-105">New WCF applications can be deployed on the same machines as existing ASP.NET Web services.</span></span> <span data-ttu-id="bae74-106">Wenn die vorhandenen ASP.NET-Webdienste eine Version von .NET Framework vor Version 2.0 verwenden, klicken Sie dann können das ASP.NET IIS Registration Tool Sie .NET Framework 2.0 auf IIS-Anwendungen bereitzustellen in der neue WCF-Anwendungen sind, gehostet werden.</span><span class="sxs-lookup"><span data-stu-id="bae74-106">If the existing ASP.NET Web services use a version of the .NET Framework prior to version 2.0, then you can use the ASP.NET IIS Registration Tool to selectively deploy the .NET Framework 2.0 to IIS applications in which new WCF applications are to be hosted.</span></span> <span data-ttu-id="bae74-107">Dieses Tool finden Sie unter [ASP.NET IIS-Registrierungstool (Aspnet_regiis.exe)](https://go.microsoft.com/fwlink/?LinkId=94687), und verfügt über eine Benutzeroberfläche, die in der IIS 6.0-Verwaltungskonsole integriert.</span><span class="sxs-lookup"><span data-stu-id="bae74-107">That tool is documented at [ASP.NET IIS Registration Tool (Aspnet_regiis.exe)](https://go.microsoft.com/fwlink/?LinkId=94687), and has a user interface built into the IIS 6.0 management console.</span></span>

<span data-ttu-id="bae74-108">WCF kann verwendet werden, um vorhandenen ASP.NET-Webdiensten neue Funktionen hinzugefügt, durch das Hinzufügen von WCF-Dienste, die so konfiguriert, dass vorhandene ASP.NET-Webdienstanwendungen in IIS im ASP.NET-Kompatibilitätsmodus ausgeführt werden kann.</span><span class="sxs-lookup"><span data-stu-id="bae74-108">WCF can be used to add new features to existing ASP.NET Web services by adding WCF services configured to run in ASP.NET compatibility mode to existing ASP.NET Web service applications in IIS.</span></span> <span data-ttu-id="bae74-109">Aufgrund der ASP.NET-Kompatibilitätsmodus, der Code für die neuen WCF-Dienste kann zugreifen und diese aktualisieren die gleichen anwendungsstatusinformationen wie der zuvor vorhandene ASP.NET-Code mithilfe der <xref:System.Web.HttpContext> Klasse.</span><span class="sxs-lookup"><span data-stu-id="bae74-109">Because of ASP.NET compatibility mode, the code for the new WCF services can access and update the same application state information as the pre-existing ASP.NET code, by using the <xref:System.Web.HttpContext> class.</span></span> <span data-ttu-id="bae74-110">Außerdem können sich die Anwendungen die gleichen Klassenbibliotheken teilen.</span><span class="sxs-lookup"><span data-stu-id="bae74-110">The applications can also share the same class libraries.</span></span>

<span data-ttu-id="bae74-111">WCF-Clients können ASP.NET-Webdienste verwenden.</span><span class="sxs-lookup"><span data-stu-id="bae74-111">WCF clients can use ASP.NET Web services.</span></span> <span data-ttu-id="bae74-112">WCF-Dienste, die mit konfiguriert werden die <xref:System.ServiceModel.BasicHttpBinding> können von ASP.NET-Webdienstclients verwendet werden.</span><span class="sxs-lookup"><span data-stu-id="bae74-112">WCF services that are configured with the <xref:System.ServiceModel.BasicHttpBinding> can be used by ASP.NET Web service clients.</span></span> <span data-ttu-id="bae74-113">ASP.NET Web Services können mit WCF-Anwendungen gemeinsam vorliegen, und WCF kann auch verwendet werden, um vorhandenen ASP.NET-Webdiensten Funktionen hinzuzufügen.</span><span class="sxs-lookup"><span data-stu-id="bae74-113">ASP.NET Web services can co-exist with WCF applications, and WCF can even be used to add features to existing ASP.NET Web services.</span></span> <span data-ttu-id="bae74-114">Wenn alle der folgenden Methoden, die in denen WCF und ASP.NET Web Services zusammen verwendet werden können, sollten Sie migrieren von ASP.NET-Webdiensten zu WCF, nur dann, wenn Sie Funktionen benötigen, die von WCF und nicht von ASP.NET Web Services bereitgestellt werden.</span><span class="sxs-lookup"><span data-stu-id="bae74-114">Given all of these ways in which WCF and ASP.NET Web services can be used together, you may want to migrate ASP.NET Web services to WCF only if you require features that are provided by WCF and not ASP.NET Web services.</span></span>

<span data-ttu-id="bae74-115">Auch in den wenigen Fällen, in denen es erforderlich ist, ist die Migrieren von Code von einer Technologie in eine andere selten der richtige Ansatz.</span><span class="sxs-lookup"><span data-stu-id="bae74-115">Even in the few cases where it is necessary, migrating code from one technology to another is seldom the correct approach.</span></span> <span data-ttu-id="bae74-116">Der Grund der Übernahme der neuen Technologie ist die Erfüllung neuer Anforderungen, die von der früheren Technologie nicht erfüllt werden können; in diesem Fall ist die richtige Vorgehensweise, eine neue Lösung zu entwerfen, die den neuen, erweiterten Satz an Anforderungen erfüllt.</span><span class="sxs-lookup"><span data-stu-id="bae74-116">The reason for adopting the new technology is to meet new requirements that cannot be met with the earlier technology, and in that case, the correct thing to do is to design a new solution to meet the newly-expanded set of requirements.</span></span> <span data-ttu-id="bae74-117">Das neue Design profitiert von Ihren Erfahrungen mit dem vorhandenen System und von den Einsichten, die seit dem Design des neuen Systems gewonnen wurden.</span><span class="sxs-lookup"><span data-stu-id="bae74-117">The new design benefits from your experience with the existing system and from wisdom gained since that system was designed.</span></span> <span data-ttu-id="bae74-118">Das neue Design kann außerdem die Fähigkeiten der neuen Technologien komplett ausreizen, anstatt das alte Design auf der neuen Plattform zu reproduzieren.</span><span class="sxs-lookup"><span data-stu-id="bae74-118">The new design can also use the full capabilities of the new technologies rather than reproducing the old design on the new platform.</span></span> <span data-ttu-id="bae74-119">Indem Schlüsselelemente des neuen Designs zu Prototypen gemacht werden, wird es einfacher, Code aus dem vorhandenen System innerhalb des neuen Systems wiederzuverwerten.</span><span class="sxs-lookup"><span data-stu-id="bae74-119">After prototyping key elements of the new design, it becomes easier to re-use code from the existing system within the new one.</span></span>
