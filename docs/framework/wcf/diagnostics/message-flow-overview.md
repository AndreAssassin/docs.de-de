### <a name="to-enable-message-flow-tracing-in-the-getting-started-tutorial"></a><span data-ttu-id="d50e7-113">So aktivieren Sie die Ablaufverfolgung des Nachrichtenflusses im Lernprogramm "Erste Schritte"</span><span class="sxs-lookup"><span data-stu-id="d50e7-113">To enable message flow tracing in the Getting Started tutorial</span></span>

1. <span data-ttu-id="d50e7-114">Ereignisanzeige öffnen, indem Sie auf **starten**, **ausführen**, und geben Sie `eventvwr.exe`.</span><span class="sxs-lookup"><span data-stu-id="d50e7-114">Open Event Viewer by clicking **Start**, **Run**, and entering `eventvwr.exe`.</span></span>

2. <span data-ttu-id="d50e7-115">Wenn Sie analytische Ablaufverfolgung nicht aktiviert haben, erweitern Sie **Anwendungs- und Dienstprotokolle**, **Microsoft**, **Windows**, **Anwendungsserver-Anwendungen** .</span><span class="sxs-lookup"><span data-stu-id="d50e7-115">If you haven’t enabled analytic tracing, expand **Applications and Services Logs**, **Microsoft**, **Windows**, **Application Server-Applications**.</span></span> <span data-ttu-id="d50e7-116">Wählen Sie **Ansicht**, **analytische und Debugprotokolle einblenden**.</span><span class="sxs-lookup"><span data-stu-id="d50e7-116">Select **View**, **Show Analytic and Debug Logs**.</span></span> <span data-ttu-id="d50e7-117">Mit der rechten Maustaste **analytisch** , und wählen Sie **Protokoll aktivieren**.</span><span class="sxs-lookup"><span data-stu-id="d50e7-117">Right-click **Analytic** and select **Enable Log**.</span></span> <span data-ttu-id="d50e7-118">Lassen Sie die Ereignisanzeige geöffnet, damit die Ablaufverfolgungen eingesehen werden können.</span><span class="sxs-lookup"><span data-stu-id="d50e7-118">Leave Event Viewer open so that traces can be viewed.</span></span>

3. <span data-ttu-id="d50e7-119">Öffnen Sie das Beispiel erstellt der [Getting Started Tutorial](../../../../docs/framework/wcf/getting-started-tutorial.md) in Visual Studio 2012.</span><span class="sxs-lookup"><span data-stu-id="d50e7-119">Open the sample created in the [Getting Started Tutorial](../../../../docs/framework/wcf/getting-started-tutorial.md) in Visual Studio 2012.</span></span> <span data-ttu-id="d50e7-120">Beachten Sie, dass Sie Visual Studio 2012 als Administrator ausführen müssen, damit der Dienst erstellt werden kann.</span><span class="sxs-lookup"><span data-stu-id="d50e7-120">Note that you must run Visual Studio 2012 as an administrator so that the service can be created.</span></span> <span data-ttu-id="d50e7-121">Wenn Sie die WCF-Beispiele installiert haben, können Sie öffnen die [Einstieg](../../../../docs/framework/wcf/samples/getting-started-sample.md), enthält das abgeschlossene Projekt, das in diesem Tutorial erstellt haben.</span><span class="sxs-lookup"><span data-stu-id="d50e7-121">If you have the WCF samples installed, you can open the [Getting Started](../../../../docs/framework/wcf/samples/getting-started-sample.md), which contains the completed project created in the tutorial.</span></span>

4. <span data-ttu-id="d50e7-122">Mit der rechten Maustaste die **Service** Projekt, und wählen **hinzufügen**, **neues Element**.</span><span class="sxs-lookup"><span data-stu-id="d50e7-122">Right-click the **Service** project and select **Add**, **New Item**.</span></span> <span data-ttu-id="d50e7-123">Wählen Sie **Anwendungskonfigurationsdatei** , und klicken Sie auf **OK**.</span><span class="sxs-lookup"><span data-stu-id="d50e7-123">Select **Application Configuration File** and click **OK**.</span></span>

5. <span data-ttu-id="d50e7-124">Fügen Sie der im vorherigen Schritt erstellten Datei App.Config den folgenden Code hinzu.</span><span class="sxs-lookup"><span data-stu-id="d50e7-124">Add the following code to the App.Config file created in the previous step.</span></span>

    ```xml
      <diagnostics>
        <endToEndTracing propagateActivity="true" messageFlowTracing="true"/>
      </diagnostics>
    </system.serviceModel>
    ```

6. <span data-ttu-id="d50e7-125">Führen Sie die Serveranwendung ohne Debuggen aus, indem Sie STRG+F5 drücken.</span><span class="sxs-lookup"><span data-stu-id="d50e7-125">Execute the server application without debugging by pressing CTRL+F5.</span></span> <span data-ttu-id="d50e7-126">Führen Sie das Clientprojekt, indem Sie mit der rechten Maustaste die **Client** Projekt- und Auswählen von **Debuggen**, **neue Instanz starten**.</span><span class="sxs-lookup"><span data-stu-id="d50e7-126">Execute the client project by right-clicking the **Client** project and selecting **Debug**, **Start New Instance**.</span></span>

7. <span data-ttu-id="d50e7-127">Um die Ereignisse vom Client zum Server zu verfolgen, fügen Sie der Anwendungskonfigurationsdatei im Projekt Client Folgendes hinzu.</span><span class="sxs-lookup"><span data-stu-id="d50e7-127">To trace the events from the client to the server, add the following to the application configuration file in the Client project.</span></span>

    ```xml
    <diagnostics>
      <endToEndTracing propagateActivity="true" messageFlowTracing="true"/>
    </diagnostics>
    ```

8. <span data-ttu-id="d50e7-128">Fügen Sie in Program.cs auf dem Client die folgende Using-Anweisung hinzu.</span><span class="sxs-lookup"><span data-stu-id="d50e7-128">In Program.cs in the client, add the following Using statement.</span></span>

    ```csharp
    using System.Diagnostics;
    ```

9. <span data-ttu-id="d50e7-129">Legen Sie in der Main-Methode in der Datei program.cs im Clientprojekt die Ablaufverfolgungs-GUID fest, die im Ereignisprotokoll weitergegeben werden soll.</span><span class="sxs-lookup"><span data-stu-id="d50e7-129">In the Main method in the program.cs file in the client project, set the Trace GUID to be propagated in the event log.</span></span>

    ```csharp

    Trace.CorrelationManager.ActivityId = guid;
    ```

10. <span data-ttu-id="d50e7-130">Aktualisieren und Überprüfen der **analytisch** Protokoll.</span><span class="sxs-lookup"><span data-stu-id="d50e7-130">Refresh and examine the **Analytic**  log.</span></span>  <span data-ttu-id="d50e7-131">Suchen Sie nach einem Ereignis mit der Ereignis-ID 220.</span><span class="sxs-lookup"><span data-stu-id="d50e7-131">Look for an event with Event ID 220.</span></span>  <span data-ttu-id="d50e7-132">Wählen Sie das Ereignis aus, und klicken Sie auf die **Details** Registerkarte im Vorschaufenster angezeigt.</span><span class="sxs-lookup"><span data-stu-id="d50e7-132">Select the event, and click the **Details** tab in the preview pane.</span></span> <span data-ttu-id="d50e7-133">Dieses Ereignis enthält die Korrelations-ID für die aufrufende Aktivität.</span><span class="sxs-lookup"><span data-stu-id="d50e7-133">This event will contain the correlation ID for the calling activity.</span></span>

    ```xml

    ```

    > [!NOTE]
    >  <span data-ttu-id="d50e7-134">Alle Ereignisse mit derselben GUID in der ActivityID beziehen sich auf eine Anforderung.</span><span class="sxs-lookup"><span data-stu-id="d50e7-134">All events with the same GUID in the ActivityID are related to one request.</span></span> <span data-ttu-id="d50e7-135">Damit können Nachrichten von einem bestimmten Client mit einem bestimmten Dienst korreliert werden.</span><span class="sxs-lookup"><span data-stu-id="d50e7-135">This can be used to correlate messages from a specific client to a specific service.</span></span> <span data-ttu-id="d50e7-136">Wenn der Client einen anderen Dienst aufgerufen hat, kann der Client anhand der ActivityID identifiziert werden.</span><span class="sxs-lookup"><span data-stu-id="d50e7-136">If the client called another service, then the same client could be identified by ActivityID.</span></span>

11. <span data-ttu-id="d50e7-137">Gelegentlich kann die ActivityID von der ursprünglichen GUID in eine neue ActivityID geändert werden.</span><span class="sxs-lookup"><span data-stu-id="d50e7-137">In some cases, the ActivityID can change from the original GUID to a new ActivityID.</span></span> <span data-ttu-id="d50e7-138">In diesem Fall wird ein Übertragungsereignis ausgegeben.</span><span class="sxs-lookup"><span data-stu-id="d50e7-138">In that case, a transfer event is emitted.</span></span> <span data-ttu-id="d50e7-139">Diese Ereignis-ID lautet 499, und das Ereignis enthält die folgenden Daten im Header.</span><span class="sxs-lookup"><span data-stu-id="d50e7-139">This event ID is 499, and the event will contain the following data in the header.</span></span>

    ```xml

        <System>
            <Provider Name="Microsoft-Windows-Application Server-Applications" Guid="{c651f5f6-1c0d-492e-8ae1-b4efd7c9d503}" />
            <EventID>499</EventID>
            ...
            <Correlation ActivityID="{A066CCF1-8AB3-459B-B62F-F79F957A5036}" RelatedActivityID="{85FC0930-9C49-42DA-804B-A7368104BD1B}" />
            ...
       </System>
    </Event>
    ```

    > [!NOTE]
    >  <span data-ttu-id="d50e7-140">Im Übertragungsereignis wird die Änderung der aktiven ActivityID von der als ActivityID angegebenen GUID in die GUID aufgezeichnet, die als RelatedActivityID angegeben ist.</span><span class="sxs-lookup"><span data-stu-id="d50e7-140">The transfer event records the change of the active ActivityID from the GUID specified as the ActivityID to the GUID specified as RelatedActivityID.</span></span> <span data-ttu-id="d50e7-141">Nach Ausgabe des Übertragungsereignisses enthalten sämtliche Ereignisse die neue GUID als ActivityID.</span><span class="sxs-lookup"><span data-stu-id="d50e7-141">After the transfer event is emitted, all events will contain the new GUID as the ActivityID.</span></span>
