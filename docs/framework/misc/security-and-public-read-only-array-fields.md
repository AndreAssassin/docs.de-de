# <a name="security-and-public-read-only-array-fields"></a><span data-ttu-id="47b72-102">Sicherheit und schreibgeschützte öffentliche Arrayfelder</span><span class="sxs-lookup"><span data-stu-id="47b72-102">Security and Public Read-only Array Fields</span></span>
<span data-ttu-id="47b72-103">Verwenden Sie niemals schreibgeschützte öffentliche Array Felder aus verwalteten Bibliotheken, um das Begrenzungs Verhalten oder die Sicherheit Ihrer Anwendungen zu definieren, da schreibgeschützte öffentliche Array Felder geändert werden können.</span><span class="sxs-lookup"><span data-stu-id="47b72-103">Never use read-only public array fields from managed libraries to define the boundary behavior or security of your applications because read-only public array fields can be modified.</span></span>  
  
## <a name="remarks"></a><span data-ttu-id="47b72-104">Bemerkungen</span><span class="sxs-lookup"><span data-stu-id="47b72-104">Remarks</span></span>  
 <span data-ttu-id="47b72-105">Einige .NET Framework-Klassen enthalten schreibgeschützte öffentliche Felder, die plattformspezifische Begrenzungs Parameter enthalten.</span><span class="sxs-lookup"><span data-stu-id="47b72-105">Some .NET framework classes include read-only public fields that contain platform-specific boundary parameters.</span></span>  <span data-ttu-id="47b72-106">Beispielsweise ist das <xref:System.IO.Path.InvalidPathChars> Feld ein Array, in dem die Zeichen beschrieben werden, die in einer Dateipfad-Zeichenfolge nicht zulässig sind.</span><span class="sxs-lookup"><span data-stu-id="47b72-106">For example, the <xref:System.IO.Path.InvalidPathChars> field is an array that describes the characters that are not allowed in a file path string.</span></span>  <span data-ttu-id="47b72-107">Viele ähnliche Felder sind im gesamten .NET Framework vorhanden.</span><span class="sxs-lookup"><span data-stu-id="47b72-107">Many similar fields are present throughout the .NET Framework.</span></span>  
  
 <span data-ttu-id="47b72-108">Die Werte von öffentlichen schreibgeschützten Feldern wie <xref:System.IO.Path.InvalidPathChars> können durch Ihren Code oder Code geändert werden, der die Anwendungsdomäne Ihres Codes freigibt.</span><span class="sxs-lookup"><span data-stu-id="47b72-108">The values of public read-only fields like <xref:System.IO.Path.InvalidPathChars> can be modified by your code or code that shares your code’s application domain.</span></span>  <span data-ttu-id="47b72-109">Verwenden Sie keine schreibgeschützten öffentlichen Array Felder wie diese, um das Begrenzungs Verhalten Ihrer Anwendungen zu definieren.</span><span class="sxs-lookup"><span data-stu-id="47b72-109">You should not use read-only public array fields like this to define the boundary behavior of your applications.</span></span>  <span data-ttu-id="47b72-110">Wenn Sie dies tun, kann bösartiger Code die Begrenzungs Definitionen ändern und Ihren Code auf unerwartete Weise verwenden.</span><span class="sxs-lookup"><span data-stu-id="47b72-110">If you do, malicious code can alter the boundary definitions and use your code in unexpected ways.</span></span>  
  
 <span data-ttu-id="47b72-111">In Version 2,0 und höher der .NET Framework sollten Sie Methoden verwenden, die ein neues Array zurückgeben, anstatt öffentliche Array Felder zu verwenden.</span><span class="sxs-lookup"><span data-stu-id="47b72-111">In version 2.0 and later of the .NET Framework, you should use methods that return a new array instead of using public array fields.</span></span>  <span data-ttu-id="47b72-112">Anstatt das <xref:System.IO.Path.InvalidPathChars>-Feld zu verwenden, sollten Sie z. b. die <xref:System.IO.Path.GetInvalidPathChars%2A>-Methode verwenden.</span><span class="sxs-lookup"><span data-stu-id="47b72-112">For example, instead of using the <xref:System.IO.Path.InvalidPathChars> field, you should use the <xref:System.IO.Path.GetInvalidPathChars%2A> method.</span></span>  
  
 <span data-ttu-id="47b72-113">Beachten Sie, dass die .NET Framework Typen die öffentlichen Felder nicht zum internen definieren von Begrenzungs Typen verwenden.</span><span class="sxs-lookup"><span data-stu-id="47b72-113">Note that the .NET Framework types do not use the public fields to define boundary types internally.</span></span>  <span data-ttu-id="47b72-114">Stattdessen verwendet der .NET Framework separate private Felder.</span><span class="sxs-lookup"><span data-stu-id="47b72-114">Instead, the .NET Framework uses separate private fields.</span></span>  <span data-ttu-id="47b72-115">Wenn Sie die Werte dieser öffentlichen Felder ändern, ändert sich das Verhalten von .NET Framework Typen nicht.</span><span class="sxs-lookup"><span data-stu-id="47b72-115">Changing the values of these public fields does not alter the behavior of .NET Framework types.</span></span>
