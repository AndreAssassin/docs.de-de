# <a name="namespaces-entity-sql"></a><span data-ttu-id="150a1-102">Namespaces (Entity SQL)</span><span class="sxs-lookup"><span data-stu-id="150a1-102">Namespaces (Entity SQL)</span></span>
[!INCLUDE[esql](../../../../../../includes/esql-md.md)] <span data-ttu-id="150a1-103">vermeidet Namenskonflikte bei globalen Bezeichnern wie Typnamen, Entitätenmengen, Funktionen usw. mithilfe von Namespaces.</span><span class="sxs-lookup"><span data-stu-id="150a1-103">introduces namespaces to avoid name conflicts for global identifiers such as type names, entity sets, functions, and so on.</span></span> <span data-ttu-id="150a1-104">Die Namespace Unterstützung in [!INCLUDE[esql](../../../../../../includes/esql-md.md)] ähnelt der Namespace Unterstützung in der .NET Framework.</span><span class="sxs-lookup"><span data-stu-id="150a1-104">The namespace support in [!INCLUDE[esql](../../../../../../includes/esql-md.md)] is similar to the namespace support in the .NET Framework.</span></span>  
  
 [!INCLUDE[esql](../../../../../../includes/esql-md.md)] <span data-ttu-id="150a1-105">stellt, wie in folgendem Beispiel veranschaulicht, zwei Arten der USING-Klausel zur Verfügung: qualifizierte Namespaces (wobei ein kürzerer Alias für den Namespace bereitgestellt wird) und unqualifizierte Namespaces:</span><span class="sxs-lookup"><span data-stu-id="150a1-105">provides two forms of the USING clause: qualified namespaces (where a shorter alias is provided for the namespace), and unqualified namespaces, as illustrated in the following example:</span></span>

## <a name="name-resolution-rules"></a><span data-ttu-id="150a1-106">Regeln zur Namensauflösung</span><span class="sxs-lookup"><span data-stu-id="150a1-106">Name Resolution Rules</span></span>  
 <span data-ttu-id="150a1-107">Wenn ein Bezeichner nicht in den lokalen Bereichen aufgelöst werden kann, versucht [!INCLUDE[esql](../../../../../../includes/esql-md.md)], den Namen in den globalen Bereichen (den Namespaces) zu finden.</span><span class="sxs-lookup"><span data-stu-id="150a1-107">If an identifier cannot be resolved in the local scopes, [!INCLUDE[esql](../../../../../../includes/esql-md.md)] tries to locate the name in the global scopes (the namespaces).</span></span> [!INCLUDE[esql](../../../../../../includes/esql-md.md)] <span data-ttu-id="150a1-108">vergleicht zunächst den Bezeichner (Präfix) mit einem der qualifizierten Namespaces.</span><span class="sxs-lookup"><span data-stu-id="150a1-108">first tries to match the identifier (prefix) with one of the qualified namespaces.</span></span> <span data-ttu-id="150a1-109">Wenn eine Entsprechung vorliegt, versucht [!INCLUDE[esql](../../../../../../includes/esql-md.md)], den Rest des Bezeichners im angegebenen Namespace aufzulösen.</span><span class="sxs-lookup"><span data-stu-id="150a1-109">If there is a match, [!INCLUDE[esql](../../../../../../includes/esql-md.md)] tries to resolve the rest of the identifier in the specified namespace.</span></span> <span data-ttu-id="150a1-110">Wenn keine Übereinstimmung festgestellt werden kann, wird eine Ausnahme ausgelöst.</span><span class="sxs-lookup"><span data-stu-id="150a1-110">If no match is found, an exception is thrown.</span></span>  
  
 <span data-ttu-id="150a1-111">Als nächstes versucht [!INCLUDE[esql](../../../../../../includes/esql-md.md)], alle nicht qualifizierten Namespaces (die im Prolog angegeben sind) für den Bezeichner zu durchsuchen.</span><span class="sxs-lookup"><span data-stu-id="150a1-111">Next, [!INCLUDE[esql](../../../../../../includes/esql-md.md)] tries to search all unqualified namespaces (specified in the prolog) for the identifier.</span></span> <span data-ttu-id="150a1-112">Wird der Bezeichner in genau einem Namespace gefunden, wird dieser Bereich zurückgegeben.</span><span class="sxs-lookup"><span data-stu-id="150a1-112">If the identifier can be located in exactly one namespace, that location is returned.</span></span> <span data-ttu-id="150a1-113">Wenn dem Bezeichner mehrere Namespaces entsprechen, wird eine Ausnahme ausgelöst.</span><span class="sxs-lookup"><span data-stu-id="150a1-113">If more than one namespace has a match for that identifier, an exception is thrown.</span></span> <span data-ttu-id="150a1-114">Wenn für den Bezeichner kein Namespace identifiziert werden kann, übergibt [!INCLUDE[esql](../../../../../../includes/esql-md.md)] den Namen an den nächsten äußeren Bereich (das <xref:System.Data.Common.DbCommand> oder <xref:System.Data.Common.DbConnection> Objekt), wie im folgenden Beispiel veranschaulicht:</span><span class="sxs-lookup"><span data-stu-id="150a1-114">If no namespace can be identified for the identifier, [!INCLUDE[esql](../../../../../../includes/esql-md.md)] passes the name onto the next outward scope (the <xref:System.Data.Common.DbCommand> or <xref:System.Data.Common.DbConnection> object), as illustrated in the following example:</span></span>  
  
```sql  
SELECT TREAT(p AS NamespaceName.Employee)  
FROM ContainerName.Person AS p  
WHERE p IS OF (NamespaceName.Employee)  
```
