# <a name="sqltypes-and-the-dataset"></a><span data-ttu-id="a16e4-102">"SqlTypes" und "DataSet"</span><span class="sxs-lookup"><span data-stu-id="a16e4-102">SqlTypes and the DataSet</span></span>
<span data-ttu-id="a16e4-103">Mit ADO.NET 2.0 wurde eine erweiterte Typunterstützung für das `DataSet` durch den <xref:System.Data.SqlTypes>-Namespace eingeführt.</span><span class="sxs-lookup"><span data-stu-id="a16e4-103">ADO.NET 2.0 introduced enhanced type support for the `DataSet` through the  <xref:System.Data.SqlTypes> namespace.</span></span> <span data-ttu-id="a16e4-104">Die Typen in <xref:System.Data.SqlTypes> stellen Datentypen mit derselben Semantik und Präzision wie die Datentypen in einer SQL Server-Datenbank bereit.</span><span class="sxs-lookup"><span data-stu-id="a16e4-104">The types in <xref:System.Data.SqlTypes> are designed to provide data types with the same semantics and precision as the data types in a SQL Server database.</span></span> <span data-ttu-id="a16e4-105">Für jeden Datentyp in <xref:System.Data.SqlTypes> gibt es einen äquivalenten Datentyp in SQL Server mit derselben zugrunde liegenden Datendarstellung.</span><span class="sxs-lookup"><span data-stu-id="a16e4-105">Each data type in <xref:System.Data.SqlTypes> has an equivalent data type in SQL Server, with the same underlying data representation.</span></span>  
  
 <span data-ttu-id="a16e4-106">Die direkte Verwendung von <xref:System.Data.SqlTypes> in einem <xref:System.Data.DataSet> beim Arbeiten mit SQL Server-Datentypen hat mehrere Vorteile.</span><span class="sxs-lookup"><span data-stu-id="a16e4-106">Using <xref:System.Data.SqlTypes> directly in a <xref:System.Data.DataSet> confers several benefits when working with SQL Server data types.</span></span> <span data-ttu-id="a16e4-107"><xref:System.Data.SqlTypes> unterstützt die gleiche Semantik wie systemeigene SQL Server-Datentypen.</span><span class="sxs-lookup"><span data-stu-id="a16e4-107"><xref:System.Data.SqlTypes> supports the same semantics as SQL Server native data types.</span></span> <span data-ttu-id="a16e4-108">Bei Angabe eines <xref:System.Data.SqlTypes> in der Definition eines <xref:System.Data.DataColumn>-Objekts wird der Verlust an Genauigkeit ausgeglichen, die auftreten kann, wenn Dezimaldatentypen oder numerische Datentypen zu einem Datentyp der Common Language Runtime (CLR) konvertiert werden.</span><span class="sxs-lookup"><span data-stu-id="a16e4-108">Specifying one of the <xref:System.Data.SqlTypes> in the definition of a <xref:System.Data.DataColumn> eliminates the loss of precision that can occur when converting decimal or numeric data types to one of the common language runtime (CLR) data types.</span></span>
