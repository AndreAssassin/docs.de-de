# <a name="entity-data-model-inheritance"></a><span data-ttu-id="fc9ea-102">Entity Data Model: Vererbung</span><span class="sxs-lookup"><span data-stu-id="fc9ea-102">Entity Data Model: Inheritance</span></span>
<span data-ttu-id="fc9ea-103">Der Entity Data Model (EDM) unterstützt die Vererbung von [Entitäts Typen](entity-type.md).</span><span class="sxs-lookup"><span data-stu-id="fc9ea-103">The Entity Data Model (EDM) supports inheritance for [entity types](entity-type.md).</span></span> <span data-ttu-id="fc9ea-104">Vererbung im EDM ist ähnlich wie Vererbung für Klassen in objektorientierten Programmiersprachen.</span><span class="sxs-lookup"><span data-stu-id="fc9ea-104">Inheritance in the EDM is similar to inheritance for classes in object-oriented programming languages.</span></span> <span data-ttu-id="fc9ea-105">Wie bei Klassen in objektorientierten Sprachen können Sie in einem konzeptionellen Modell einen Entitätstyp (einen *abgeleiteten Typ*) definieren, der von einem anderen Entitätstyp (dem *Basistyp*) erbt.</span><span class="sxs-lookup"><span data-stu-id="fc9ea-105">Like with classes in object-oriented languages, in a conceptual model you can define an entity type (a *derived type*) that inherits from another entity type (the *base type*).</span></span> <span data-ttu-id="fc9ea-106">Im Gegensatz zu Klassen in der objektorientierten Programmierung erbt der abgeleitete Typ in einem konzeptionellen Modell immer alle [Eigenschaften](property.md) und [Navigations Eigenschaften](navigation-property.md) des Basistyps.</span><span class="sxs-lookup"><span data-stu-id="fc9ea-106">However, unlike classes in object-oriented programming, in a conceptual model the derived type always inherits all the [properties](property.md) and [navigation properties](navigation-property.md) of the base type.</span></span> <span data-ttu-id="fc9ea-107">Geerbte Eigenschaften in einem abgeleiteten Typ können nicht überschrieben werden.</span><span class="sxs-lookup"><span data-stu-id="fc9ea-107">You cannot override inherited properties in a derived type.</span></span>  
  
 <span data-ttu-id="fc9ea-108">In einem konzeptionellen Modell können Sie Vererbungshierarchien erstellen, in denen ein abgeleiteter Typ von einem anderen abgeleiteten Typ erbt.</span><span class="sxs-lookup"><span data-stu-id="fc9ea-108">In a conceptual model you can build inheritance hierarchies in which a derived type inherits from another derived type.</span></span> <span data-ttu-id="fc9ea-109">Der Typ am oberen Rand der Hierarchie (der einzige Typ in der Hierarchie, der kein abgeleiteter Typ ist) wird als *Stammtyp*bezeichnet.</span><span class="sxs-lookup"><span data-stu-id="fc9ea-109">The type at the top of the hierarchy (the one type in the hierarchy that is not a derived type) is called the *root type*.</span></span> <span data-ttu-id="fc9ea-110">In einer Vererbungs Hierarchie muss der [Entitäts Schlüssel](entity-key.md) für den Stammtyp definiert werden.</span><span class="sxs-lookup"><span data-stu-id="fc9ea-110">In an inheritance hierarchy, the [entity key](entity-key.md) must be defined on the root type.</span></span>  
  
 <span data-ttu-id="fc9ea-111">Sie können keine Vererbungshierarchien erstellen, in denen ein abgeleiteter Typ von mehr als einem Typ erbt.</span><span class="sxs-lookup"><span data-stu-id="fc9ea-111">You cannot build inheritance hierarchies in which a derived type inherits from more than one type.</span></span> <span data-ttu-id="fc9ea-112">In einem konzeptionellen Modell mit einem `Book`-Entitätstyp könnten Sie z. B. die abgeleiteten Typen `FictionBook` und `NonFictionBook` definieren, die jeweils von `Book` erben.</span><span class="sxs-lookup"><span data-stu-id="fc9ea-112">For example, in a conceptual model with a `Book` entity type, you could define derived types `FictionBook` and `NonFictionBook` that each inherit from `Book`.</span></span> <span data-ttu-id="fc9ea-113">Sie könnten dann jedoch keinen Typ definieren, der vom Typ `FictionBook` und vom Typ `NonFictionBook` erbt.</span><span class="sxs-lookup"><span data-stu-id="fc9ea-113">However, you could not then define a type that inherits from both the `FictionBook` and `NonFictionBook` types.</span></span>
