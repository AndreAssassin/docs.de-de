# <a name="argument-not-optional-visual-basic"></a>Das Argument ist nicht optional (Visual Basic)

Die Anzahl und die Typen der Argumente m√ºssen mit den erwarteten Typen identisch sein. Entweder gibt es eine falsche Anzahl von Argumenten, oder ein ausgelassenes Argument ist nicht optional. Ein Argument kann nur beim Abrufen einer benutzerdefinierten Prozedur ausgelassen werden, wenn es in der Prozedur Definition als `Optional` deklariert wurde.  
  
## <a name="to-correct-this-error"></a>So beheben Sie diesen Fehler  
  
1. Stellen Sie alle erforderlichen Argumente bereit.  
  
2. Stellen Sie sicher, dass ausgelassene Argumente optional sind. Wenn dies nicht der Fall ist, geben Sie entweder das-Argument im-Befehl an, oder deklarieren Sie den Parameter `Optional` in der Definition.
