## <a name="xml-names"></a><span data-ttu-id="95255-104">XML-Namen</span><span class="sxs-lookup"><span data-stu-id="95255-104">XML Names</span></span>

 <span data-ttu-id="95255-111">Ein anderer Aspekt von XML-Namen sind die XML-*Namespacepräfixe*.</span><span class="sxs-lookup"><span data-stu-id="95255-111">Another aspect of XML names is XML *namespace prefixes*.</span></span> <span data-ttu-id="95255-112">XML-Präfixe sind die Hauptursache für die Komplexität von XML-Namen.</span><span class="sxs-lookup"><span data-stu-id="95255-112">XML prefixes cause most of the complexity of XML names.</span></span> <span data-ttu-id="95255-113">Diese Präfixe ermöglichen es Ihnen, eine Kurzform eines XML-Namespace zu erstellen, wodurch das XML-Dokument kompakter und verständlicher wird.</span><span class="sxs-lookup"><span data-stu-id="95255-113">These prefixes enable you to create a shortcut for an XML namespace, which makes the XML document more concise and understandable.</span></span> <span data-ttu-id="95255-114">Die Bedeutung der XML-Präfixe ist jedoch kontextabhängig, was zur erhöhten Komplexität beiträgt.</span><span class="sxs-lookup"><span data-stu-id="95255-114">However, XML prefixes depend on their context to have meaning, which adds complexity.</span></span> <span data-ttu-id="95255-115">Das XML-Präfix `aw` kann z. B. in unterschiedlichen Teilen einer XML-Struktur unterschiedlichen XML-Namespaces zugeordnet sein.</span><span class="sxs-lookup"><span data-stu-id="95255-115">For example, the XML prefix `aw` could be associated with one XML namespace in one part of an XML tree, and with a different XML namespace in a different part of the XML tree.</span></span>  
  
 <span data-ttu-id="95255-116">Bei Verwendung [!INCLUDE[sqltecxlinq](~/includes/sqltecxlinq-md.md)] mit Visual Basic und XML-Literale, müssen Sie Namespacepräfixe verwenden, bei der Arbeit mit Dokumenten in Namespaces.</span><span class="sxs-lookup"><span data-stu-id="95255-116">When using [!INCLUDE[sqltecxlinq](~/includes/sqltecxlinq-md.md)] with Visual Basic and XML literals, you must use namespace prefixes when working with documents in namespaces.</span></span>  
  
 <span data-ttu-id="95255-117">In [!INCLUDE[sqltecxlinq](~/includes/sqltecxlinq-md.md)] werden die XML-Namen in der <xref:System.Xml.Linq.XName>-Klasse dargestellt.</span><span class="sxs-lookup"><span data-stu-id="95255-117">In [!INCLUDE[sqltecxlinq](~/includes/sqltecxlinq-md.md)], the class that represents XML names is <xref:System.Xml.Linq.XName>.</span></span> <span data-ttu-id="95255-118">XML-Namen treten in der gesamten [!INCLUDE[sqltecxlinq](~/includes/sqltecxlinq-md.md)]-API häufig auf, und immer dann, wenn ein XML-Name erforderlich ist, treffen Sie auf einen <xref:System.Xml.Linq.XName>-Parameter.</span><span class="sxs-lookup"><span data-stu-id="95255-118">XML names appear frequently throughout the [!INCLUDE[sqltecxlinq](~/includes/sqltecxlinq-md.md)] API, and wherever an XML name is required, you will find an <xref:System.Xml.Linq.XName> parameter.</span></span> <span data-ttu-id="95255-119">Sie arbeiten jedoch selten direkt mit einem <xref:System.Xml.Linq.XName>-Objekt.</span><span class="sxs-lookup"><span data-stu-id="95255-119">However, you rarely work directly with an <xref:System.Xml.Linq.XName>.</span></span> <span data-ttu-id="95255-120"><xref:System.Xml.Linq.XName> enthält eine implizite Umwandlung aus einer Zeichenfolge.</span><span class="sxs-lookup"><span data-stu-id="95255-120"><xref:System.Xml.Linq.XName> contains an implicit conversion from string.</span></span>  
  
 <span data-ttu-id="95255-121">Weitere Informationen finden Sie unter <xref:System.Xml.Linq.XNamespace> und <xref:System.Xml.Linq.XName>.</span><span class="sxs-lookup"><span data-stu-id="95255-121">For more information, see <xref:System.Xml.Linq.XNamespace> and <xref:System.Xml.Linq.XName>.</span></span>
