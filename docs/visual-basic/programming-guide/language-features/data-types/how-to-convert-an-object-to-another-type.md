## <a name="example"></a><span data-ttu-id="b6060-104">Beispiel</span><span class="sxs-lookup"><span data-stu-id="b6060-104">Example</span></span>  
 <span data-ttu-id="b6060-105">Im folgenden Beispiel wird ein `Object` Variable eine `Integer` und `String`.</span><span class="sxs-lookup"><span data-stu-id="b6060-105">The following example converts an `Object` variable to an `Integer` and a `String`.</span></span>  
  
```  
Public Sub objectConversion(ByVal anObject As Object)  
    Dim anInteger As Integer  
    Dim aString As String  
    anInteger = CType(anObject, Integer)  
    aString = CType(anObject, String)  
End Sub  
```  
  
 <span data-ttu-id="b6060-106">Wenn Sie wissen, dass der Inhalt des ein `Object` Variablen sind eines bestimmten Datentyps, es ist besser, die die Variable in diesen Datentyp zu konvertieren.</span><span class="sxs-lookup"><span data-stu-id="b6060-106">If you know that the contents of an `Object` variable are of a particular data type, it is better to convert the variable to that data type.</span></span> <span data-ttu-id="b6060-107">Wenn Sie weiterhin die `Object` Variablen, es fallen entweder *Boxing* und *unboxing* (für einen Werttyp) oder *späte Bindung* (für einen Verweistyp).</span><span class="sxs-lookup"><span data-stu-id="b6060-107">If you continue to use the `Object` variable, you incur either *boxing* and *unboxing* (for a value type) or *late binding* (for a reference type).</span></span> <span data-ttu-id="b6060-108">Diese Vorgänge alle zusätzlichen Ausführung Zeit in Anspruch nehmen, und stellen die Leistung langsamer.</span><span class="sxs-lookup"><span data-stu-id="b6060-108">These operations all take extra execution time and make your performance slower.</span></span>
