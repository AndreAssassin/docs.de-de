<span data-ttu-id="77091-103">Ein *Feld* ist eine Variable eines beliebigen Typs, die direkt in einer [class](../../language-reference/keywords/class.md) oder [struct](../../language-reference/keywords/struct.md) deklariert ist.</span><span class="sxs-lookup"><span data-stu-id="77091-103">A *field* is a variable of any type that is declared directly in a [class](../../language-reference/keywords/class.md) or [struct](../../language-reference/keywords/struct.md).</span></span> <span data-ttu-id="77091-104">Felder sind *Member* Ihres enthaltenden Typs.</span><span class="sxs-lookup"><span data-stu-id="77091-104">Fields are *members* of their containing type.</span></span>  
  
 <span data-ttu-id="77091-105">Eine Klasse oder Struktur kann möglicherweise über Instanzenfelder, statische Felder oder beides verfügen.</span><span class="sxs-lookup"><span data-stu-id="77091-105">A class or struct may have instance fields or static fields or both.</span></span> <span data-ttu-id="77091-106">Instanzenfelder sind für eine Instanz eines Typs spezifisch.</span><span class="sxs-lookup"><span data-stu-id="77091-106">Instance fields are specific to an instance of a type.</span></span> <span data-ttu-id="77091-107">Wenn Sie eine Klasse T mit einem Instanzenfeld F haben, können Sie zwei Objekte vom Typ T erstellen und den Wert von F in jedem Objekt ändern, ohne dabei den Wert in dem anderen Objekt zu beeinflussen.</span><span class="sxs-lookup"><span data-stu-id="77091-107">If you have a class T, with an instance field F, you can create two objects of type T, and modify the value of F in each object without affecting the value in the other object.</span></span> <span data-ttu-id="77091-108">Im Gegensatz dazu gehört ein statisches Feld zur Klasse selbst und wird in allen Instanzen dieser Klasse gemeinsam verwendet.</span><span class="sxs-lookup"><span data-stu-id="77091-108">By contrast, a static field belongs to the class itself, and is shared among all instances of that class.</span></span> <span data-ttu-id="77091-109">Änderungen, die von der Instanz A vorgenommen wurden, werden für die Instanzen B und C direkt sichtbar, sobald sie auf das Feld zugreifen.</span><span class="sxs-lookup"><span data-stu-id="77091-109">Changes made from instance A will be visible immediately to instances B and C if they access the field.</span></span>  
  
 <span data-ttu-id="77091-110">Im Allgemeinen sollten Sie Felder nur für Variablen verwenden, die private oder geschützte Zugriffsmöglichkeiten haben.</span><span class="sxs-lookup"><span data-stu-id="77091-110">Generally, you should use fields only for variables that have private or protected accessibility.</span></span> <span data-ttu-id="77091-111">Daten, die Ihre Klasse für Clientcode verfügbar macht, sollten über [Methoden](./methods.md), [Eigenschaften](./properties.md) und [Indexer](../indexers/index.md) bereitgestellt werden.</span><span class="sxs-lookup"><span data-stu-id="77091-111">Data that your class exposes to client code should be provided through [methods](./methods.md), [properties](./properties.md) and [indexers](../indexers/index.md).</span></span> <span data-ttu-id="77091-112">Sie können sich mit diesen Konstrukten für indirekten Zugriff auf interne Felder vor ungültigen Eingabewerten schützen.</span><span class="sxs-lookup"><span data-stu-id="77091-112">By using these constructs for indirect access to internal fields, you can guard against invalid input values.</span></span> <span data-ttu-id="77091-113">Ein privates Feld, das über eine öffentliche Eigenschaft bereitgestellte Daten speichert, wird als *Sicherungsspeicher* oder *Unterstützungsfeld* bezeichnet.</span><span class="sxs-lookup"><span data-stu-id="77091-113">A private field that stores the data exposed by a public property is called a *backing store* or *backing field*.</span></span>  
  
 <span data-ttu-id="77091-114">Felder speichern in der Regel die Daten, die über Zugriff auf mehr als eine Klassenmethode verfügen müssen und länger als die Lebensdauer einer einzelnen Methode gespeichert werden müssen.</span><span class="sxs-lookup"><span data-stu-id="77091-114">Fields typically store the data that must be accessible to more than one class method and must be stored for longer than the lifetime of any single method.</span></span> <span data-ttu-id="77091-115">Z.B. verfügt eine Klasse, die ein Kalenderdatum darstellt, möglicherweise über drei Felder: Jeweils eines für Monat, Tag und Jahr.</span><span class="sxs-lookup"><span data-stu-id="77091-115">For example, a class that represents a calendar date might have three integer fields: one for the month, one for the day, and one for the year.</span></span> <span data-ttu-id="77091-116">Variablen, die außerhalb des Bereichs einer einzelnen Methode nicht verwendet werden, sollten als *lokale Variablen* innerhalb des Methodentexts selbst deklariert werden.</span><span class="sxs-lookup"><span data-stu-id="77091-116">Variables that are not used outside the scope of a single method should be declared as *local variables* within the method body itself.</span></span>
