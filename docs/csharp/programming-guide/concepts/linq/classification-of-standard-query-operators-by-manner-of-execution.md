# <a name="classification-of-standard-query-operators-by-manner-of-execution-c"></a><span data-ttu-id="c9725-102">Klassifizierung von Standardabfrageoperatoren nach Ausführungsarten (C#)</span><span class="sxs-lookup"><span data-stu-id="c9725-102">Classification of Standard Query Operators by Manner of Execution (C#)</span></span>
<span data-ttu-id="c9725-103">Die LINQ to Objects-Implementierungen des Standardabfrageoperators werden mit einer von zwei möglichen Arten ausgeführt: direkt oder zurückgestellt.</span><span class="sxs-lookup"><span data-stu-id="c9725-103">The LINQ to Objects implementations of the standard query operator methods execute in one of two main ways: immediate or deferred.</span></span> <span data-ttu-id="c9725-104">Abfrageoperatoren, die die verzögerte Ausführung verwenden, können darüber hinaus in zwei Kategorien unterteilt werden: Streaming und Nicht-Streaming.</span><span class="sxs-lookup"><span data-stu-id="c9725-104">The query operators that use deferred execution can be additionally divided into two categories: streaming and non-streaming.</span></span> <span data-ttu-id="c9725-105">Wenn Sie wissen, wie die einzelnen Abfrageoperatoren ausgeführt werden, erleichtert dies das Verständnis der Ergebnisse, die Sie von einer Abfrage erhalten.</span><span class="sxs-lookup"><span data-stu-id="c9725-105">If you know how the different query operators execute, it may help you understand the results that you get from a given query.</span></span> <span data-ttu-id="c9725-106">Dies ist insbesondere dann der Fall, wenn die Datenquelle geändert wird oder wenn Sie eine Abfrage auf Grundlage einer anderen Abfrage erstellen.</span><span class="sxs-lookup"><span data-stu-id="c9725-106">This is especially true if the data source is changing or if you are building a query on top of another query.</span></span> <span data-ttu-id="c9725-107">In diesem Thema werden die Standardabfrageoperatoren gemäß ihrer Ausführungsarten klassifiziert.</span><span class="sxs-lookup"><span data-stu-id="c9725-107">This topic classifies the standard query operators according to their manner of execution.</span></span>  
  
## <a name="manners-of-execution"></a><span data-ttu-id="c9725-108">Arten der Ausführung</span><span class="sxs-lookup"><span data-stu-id="c9725-108">Manners of Execution</span></span>  
  
### <a name="immediate"></a><span data-ttu-id="c9725-109">Direkt</span><span class="sxs-lookup"><span data-stu-id="c9725-109">Immediate</span></span>  
 <span data-ttu-id="c9725-110">Direkte Ausführung bedeutet, dass die Datenquelle gelesen wird und die Operation an dem Zeitpunkt im Code ausgeführt wird, an dem die Abfrage deklariert wird.</span><span class="sxs-lookup"><span data-stu-id="c9725-110">Immediate execution means that the data source is read and the operation is performed at the point in the code where the query is declared.</span></span> <span data-ttu-id="c9725-111">Alle Standardabfrageoperatoren, die ein einzelnes, nicht aufzählbares Ergebnis zurückgeben, werden sofort ausgeführt.</span><span class="sxs-lookup"><span data-stu-id="c9725-111">All the standard query operators that return a single, non-enumerable result execute immediately.</span></span>  
  
### <a name="deferred"></a><span data-ttu-id="c9725-112">Zurückgestellt</span><span class="sxs-lookup"><span data-stu-id="c9725-112">Deferred</span></span>  
 <span data-ttu-id="c9725-113">Zurückgestellte Ausführung bedeutet, dass der Vorgang nicht zum Zeitpunkt im Code ausgeführt wird, an dem die Abfrage deklariert wird.</span><span class="sxs-lookup"><span data-stu-id="c9725-113">Deferred execution means that the operation is not performed at the point in the code where the query is declared.</span></span> <span data-ttu-id="c9725-114">Der Vorgang erfolgt nur, wenn die Abfragevariable aufgezählt wird, z.B. durch Verwendung einer `foreach`-Anweisung.</span><span class="sxs-lookup"><span data-stu-id="c9725-114">The operation is performed only when the query variable is enumerated, for example by using a `foreach` statement.</span></span> <span data-ttu-id="c9725-115">Dies bedeutet, dass die Ergebnisse der Ausführung der Abfrage vom Inhalt der Datenquelle zum Zeitpunkt der Abfrageausführung, nicht der Abfragedefinition abhängen.</span><span class="sxs-lookup"><span data-stu-id="c9725-115">This means that the results of executing the query depend on the contents of the data source when the query is executed rather than when the query is defined.</span></span> <span data-ttu-id="c9725-116">Wenn die Abfragevariable mehrfach aufgezählt wird, können die Ergebnisse jedes Mal abweichen.</span><span class="sxs-lookup"><span data-stu-id="c9725-116">If the query variable is enumerated multiple times, the results might differ every time.</span></span> <span data-ttu-id="c9725-117">Fast alle Standardabfrageoperatoren, deren Rückgabetyp <xref:System.Collections.Generic.IEnumerable%601> oder <xref:System.Linq.IOrderedEnumerable%601> ist, werden verzögert ausgeführt.</span><span class="sxs-lookup"><span data-stu-id="c9725-117">Almost all the standard query operators whose return type is <xref:System.Collections.Generic.IEnumerable%601> or <xref:System.Linq.IOrderedEnumerable%601> execute in a deferred manner.</span></span>  
  
 <span data-ttu-id="c9725-118">Abfrageoperatoren, die die verzögerte Ausführung verwenden, können zusätzlich als Streaming und Nicht-Streaming klassifiziert werden.</span><span class="sxs-lookup"><span data-stu-id="c9725-118">Query operators that use deferred execution can be additionally classified as streaming or non-streaming.</span></span>  
  
#### <a name="streaming"></a><span data-ttu-id="c9725-119">Streaming</span><span class="sxs-lookup"><span data-stu-id="c9725-119">Streaming</span></span>  
 <span data-ttu-id="c9725-120">Streaming-Operatoren müssen nicht alle Quelldaten lesen, bevor sie Elemente liefern.</span><span class="sxs-lookup"><span data-stu-id="c9725-120">Streaming operators do not have to read all the source data before they yield elements.</span></span> <span data-ttu-id="c9725-121">Zum Zeitpunkt der Ausführung führt ein Streaming-Operator seine Operation auf jedem Quellelement aus, während es gelesen wird, und liefert ggf. die Elemente.</span><span class="sxs-lookup"><span data-stu-id="c9725-121">At the time of execution, a streaming operator performs its operation on each source element as it is read and yields the element if appropriate.</span></span> <span data-ttu-id="c9725-122">Ein Streaming-Operator liest weiterhin Quellelemente, bis ein Ergebniselement erzeugt werden kann.</span><span class="sxs-lookup"><span data-stu-id="c9725-122">A streaming operator continues to read source elements until a result element can be produced.</span></span> <span data-ttu-id="c9725-123">Dies bedeutet, dass mehr als ein Quellelement womöglich gelesen werden kann, um ein Ergebniselement zu erzeugen.</span><span class="sxs-lookup"><span data-stu-id="c9725-123">This means that more than one source element might be read to produce one result element.</span></span>  
  
#### <a name="non-streaming"></a><span data-ttu-id="c9725-124">Nicht-Streaming</span><span class="sxs-lookup"><span data-stu-id="c9725-124">Non-Streaming</span></span>  
 <span data-ttu-id="c9725-125">Nicht-Streaming-Operatoren müssen alle Quelldaten lesen, bevor sie ein Ergebniselement liefern können.</span><span class="sxs-lookup"><span data-stu-id="c9725-125">Non-streaming operators must read all the source data before they can yield a result element.</span></span> <span data-ttu-id="c9725-126">Vorgänge wie das Sortieren oder Gruppieren fallen unter diese Kategorie.</span><span class="sxs-lookup"><span data-stu-id="c9725-126">Operations such as sorting or grouping fall into this category.</span></span> <span data-ttu-id="c9725-127">Zum Zeitpunkt der Ausführung lesen Nicht-Streaming-Operatoren alle Quelldaten, fügen sie in eine Datenstruktur ein, führen den Vorgang aus und liefern die Elemente, die sich ergeben.</span><span class="sxs-lookup"><span data-stu-id="c9725-127">At the time of execution, non-streaming query operators read all the source data, put it into a data structure, perform the operation, and yield the resulting elements.</span></span>
