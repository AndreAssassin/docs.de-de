# <a name="nullable-reference-types"></a><span data-ttu-id="fea01-104">Nullwerte zulassende Verweistypen</span><span class="sxs-lookup"><span data-stu-id="fea01-104">Nullable reference types</span></span>

<span data-ttu-id="fea01-105">Mit den in C# 8.0 eingeführten **Nullable-Verweistypen** und **Nicht-Nullable-Verweistypen** können Sie wichtige Anweisungen zu den Eigenschaften für Verweistypvariablen erstellen:</span><span class="sxs-lookup"><span data-stu-id="fea01-105">C# 8.0 introduces **nullable reference types** and **non-nullable reference types** that enable you to make important statements about the properties for reference type variables:</span></span>

- <span data-ttu-id="fea01-106">**Ein Verweis darf nicht NULL sein**.</span><span class="sxs-lookup"><span data-stu-id="fea01-106">**A reference is not supposed to be null**.</span></span> <span data-ttu-id="fea01-107">Wenn Variablen nicht NULL sein dürfen, erzwingt der Compiler Regeln, die sicherstellen, dass das Dereferenzieren dieser Variablen sicher ist, ohne zuerst zu überprüfen, dass sie nicht NULL sind:</span><span class="sxs-lookup"><span data-stu-id="fea01-107">When variables aren't supposed to be null, the compiler enforces rules that ensure it is safe to dereference these variables without first checking that it isn't null:</span></span>
  - <span data-ttu-id="fea01-108">Die Variable muss mit einem Wert ungleich NULL initialisiert werden.</span><span class="sxs-lookup"><span data-stu-id="fea01-108">The variable must be initialized to a non-null value.</span></span>
  - <span data-ttu-id="fea01-109">Der Variablen kann nie der Wert `null` zugewiesen werden.</span><span class="sxs-lookup"><span data-stu-id="fea01-109">The variable can never be assigned the value `null`.</span></span>
- <span data-ttu-id="fea01-110">**Ein Verweis darf NULL sein**.</span><span class="sxs-lookup"><span data-stu-id="fea01-110">**A reference may be null**.</span></span> <span data-ttu-id="fea01-111">Wenn Variablen NULL sein dürfen, erzwingt der Compiler verschiedene Regeln, um sicherzustellen, dass Sie richtig auf einen Nullverweis überprüft haben:</span><span class="sxs-lookup"><span data-stu-id="fea01-111">When variables may be null, the compiler enforces different rules to ensure that you've correctly checked for a null reference:</span></span>
  - <span data-ttu-id="fea01-112">Die Variable kann nur dereferenziert werden, wenn der Compiler garantieren kann, dass der Wert nicht NULL ist.</span><span class="sxs-lookup"><span data-stu-id="fea01-112">The variable may only be dereferenced when the compiler can guarantee that the value isn't null.</span></span>
  - <span data-ttu-id="fea01-113">Diese Variablen können mit dem Standardwert `null` initialisiert und in anderem Code dem Wert `null` zugewiesen werden.</span><span class="sxs-lookup"><span data-stu-id="fea01-113">These variables may be initialized with the default `null` value and may be assigned the value `null` in other code.</span></span>

<span data-ttu-id="fea01-114">Dieses neue Feature bietet große Vorteile hinsichtlich der Verarbeitung von Verweisvariablen in früheren Versionen von C#, bei denen die Entwurfsabsicht nicht über die Variablendeklaration bestimmt werden konnte.</span><span class="sxs-lookup"><span data-stu-id="fea01-114">This new feature provides significant benefits over the handling of reference variables in earlier versions of C# where the design intent couldn't be determined from the variable declaration.</span></span> <span data-ttu-id="fea01-115">Der Compiler bot keinen Schutz gegen Nullverweisausnahmen für Verweistypen:</span><span class="sxs-lookup"><span data-stu-id="fea01-115">The compiler didn't provide safety against null reference exceptions for reference types:</span></span>

- <span data-ttu-id="fea01-116">**Ein Verweis kann NULL sein**.</span><span class="sxs-lookup"><span data-stu-id="fea01-116">**A reference can be null**.</span></span> <span data-ttu-id="fea01-117">Wenn ein Verweistyp mit NULL initialisiert oder später NULL zugewiesen wird, werden keine Warnungen ausgegeben.</span><span class="sxs-lookup"><span data-stu-id="fea01-117">No warnings are issued when a reference type is initialized to null, or later assigned to null.</span></span>
- <span data-ttu-id="fea01-118">**Es wird angenommen, dass ein Verweis nicht NULL ist**.</span><span class="sxs-lookup"><span data-stu-id="fea01-118">**A reference is assumed to be not null**.</span></span> <span data-ttu-id="fea01-119">Der Compiler gibt keine Warnungen aus, wenn Verweistypen dereferenziert werden.</span><span class="sxs-lookup"><span data-stu-id="fea01-119">The compiler doesn't issue any warnings when reference types are dereferenced.</span></span> <span data-ttu-id="fea01-120">(Bei Nullable-Verweisen gibt der Compiler Warnungen aus, sobald Sie eine Variable dereferenzieren, die möglicherweise NULL ist).</span><span class="sxs-lookup"><span data-stu-id="fea01-120">(With nullable references,  the compiler issues warnings whenever you dereference a variable that may be null).</span></span>

<span data-ttu-id="fea01-121">Durch das Hinzufügen von Nullable-Verweistypen können Sie Ihre Absicht klarer deklarieren.</span><span class="sxs-lookup"><span data-stu-id="fea01-121">With the addition of nullable reference types, you can declare your intent more clearly.</span></span> <span data-ttu-id="fea01-122">Der `null`-Wert ist die richtige Wahl, um darzustellen, dass eine Variable auf keinen Wert verweist.</span><span class="sxs-lookup"><span data-stu-id="fea01-122">The `null` value is the correct way to represent that a variable doesn't refer to a value.</span></span> <span data-ttu-id="fea01-123">Verwenden Sie dieses Feature nicht dazu, alle `null`-Werte aus Ihrem Code zu entfernen.</span><span class="sxs-lookup"><span data-stu-id="fea01-123">Don't use this feature to remove all `null` values from your code.</span></span> <span data-ttu-id="fea01-124">Stattdessen sollten Sie Ihre Absicht gegenüber dem Compiler und anderen Entwicklern deklarieren, die Ihren Code lesen.</span><span class="sxs-lookup"><span data-stu-id="fea01-124">Rather, you should declare your intent to the compiler and other developers that read your code.</span></span> <span data-ttu-id="fea01-125">Indem Sie Ihre Absicht deklarieren, werden Sie vom Compiler informiert, sobald Sie Code schreiben, der dieser Absicht widerspricht.</span><span class="sxs-lookup"><span data-stu-id="fea01-125">By declaring your intent, the compiler informs you when you write code that is inconsistent with that intent.</span></span>

<span data-ttu-id="fea01-126">Ein **Nullable-Verweistyp** wird mithilfe der gleichen Syntax wie [Nullable-Werttypen](programming-guide/nullable-types/index.md) aufgeführt: Ein `?` wird an den Variablentyp angefügt.</span><span class="sxs-lookup"><span data-stu-id="fea01-126">A **nullable reference type** is noted using the same syntax as [nullable value types](programming-guide/nullable-types/index.md): a `?` is appended to the type of the variable.</span></span> <span data-ttu-id="fea01-127">Beispielsweise stellt die folgende Variablendeklaration eine Nullable-Zeichenfolgenvariable, `name`, dar:</span><span class="sxs-lookup"><span data-stu-id="fea01-127">For example, the following variable declaration represents a nullable string variable, `name`:</span></span>


<span data-ttu-id="fea01-128">Bei jeder Variable, bei der `?` nicht an den Typnamen angefügt ist, handelt es sich um einen **Nicht-Nullable-Verweistyp**.</span><span class="sxs-lookup"><span data-stu-id="fea01-128">Any variable where the `?` is not appended to the type name is a **non-nullable reference type**.</span></span> <span data-ttu-id="fea01-129">Dies umfasst alle Verweistypvariablen in vorhandenem Code, wenn Sie dieses Feature aktiviert haben.</span><span class="sxs-lookup"><span data-stu-id="fea01-129">That includes all reference type variables in existing code when you have enabled this feature.</span></span>

<span data-ttu-id="fea01-130">Der Compiler verwendet die statische Analyse, um zu bestimmen, ob ein Nullable-Verweis nicht NULL ist.</span><span class="sxs-lookup"><span data-stu-id="fea01-130">The compiler uses static analysis to determine if a nullable reference is known to be non-null.</span></span> <span data-ttu-id="fea01-131">Der Compiler warnt Sie, falls Sie einen Nullable-Verweis dereferenzieren, wenn dieser möglicherweise NULL ist.</span><span class="sxs-lookup"><span data-stu-id="fea01-131">The compiler warns you if you dereference a nullable reference when it may be null.</span></span> <span data-ttu-id="fea01-132">Sie können dieses Verhalten überschreiben, indem Sie den **NULL-toleranten Operator** (`!`) gefolgt von einem Variablennamen verwenden.</span><span class="sxs-lookup"><span data-stu-id="fea01-132">You can override this behavior by using the **null-forgiving operator** (`!`) following a variable name.</span></span> <span data-ttu-id="fea01-133">Wenn Sie beispielsweise wissen, dass die Variable `name` nicht NULL ist, der Compiler aber eine Warnung ausgibt, können Sie folgenden Code schreiben, um die Compileranalyse zu überschreiben:</span><span class="sxs-lookup"><span data-stu-id="fea01-133">For example, if you know the `name` variable isn't null but the compiler issues a warning, you can write the following code to override the compiler's analysis:</span></span>
