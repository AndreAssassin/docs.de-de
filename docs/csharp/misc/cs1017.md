# <a name="compiler-error-cs1017"></a>Compilerfehler CS1017
Catch-Klauseln können nicht auf die allgemeine catch-Klausel einer try-Anweisung folgen.  
  
 Ein `catch` -Block, der keine Parameter annimmt, muss der letzte in einer Reihe von `catch` -Blöcken sein. Weitere Informationen zu Ausnahmen finden Sie unter [Ausnahmen und Ausnahmebehandlung](../../csharp/programming-guide/exceptions/index.md).  
  
## <a name="example"></a>Beispiel  
 Im folgenden Beispiel wird CS1017 generiert:  
  
```csharp  
// CS1017.cs  
using System;  
  
namespace x  
{  
    public class b : Exception  
    {  
    }  
  
    public class a  
    {  
        public static void Main()  
        {  
            try  
            {  
            }  
  
            catch   // CS1017, must be last catch  
            {  
            }  
  
            catch(b)  
            {  
                throw;  
            }  
        }  
    }  
}  
```
