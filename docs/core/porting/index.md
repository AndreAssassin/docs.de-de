## <a name="overview-of-the-porting-process"></a><span data-ttu-id="88ea0-108">Übersicht über den Portiervorgang</span><span class="sxs-lookup"><span data-stu-id="88ea0-108">Overview of the porting process</span></span>

<span data-ttu-id="88ea0-109">Wir empfehlen die folgende Vorgehensweise, um Ihr Projekt zu .NET Core zu portieren:</span><span class="sxs-lookup"><span data-stu-id="88ea0-109">We recommend you to use the following process when porting your project to .NET Core:</span></span>

1. <span data-ttu-id="88ea0-110">Legen Sie für alle zu portierenden Projekte .NET Framework 4.7.2 oder höher als neues Ziel fest.</span><span class="sxs-lookup"><span data-stu-id="88ea0-110">Retarget all projects you wish to port to target the .NET Framework 4.7.2 or higher.</span></span>

   <span data-ttu-id="88ea0-111">Durch diesen Schritt wird sichergestellt, dass Sie API-Alternativen für bestimmte .NET Framework-Ziele verwenden können, falls .NET Core eine bestimmte API nicht unterstützt.</span><span class="sxs-lookup"><span data-stu-id="88ea0-111">This step ensures that you can use API alternatives for .NET Framework-specific targets when .NET Core doesn't support a particular API.</span></span>

2. <span data-ttu-id="88ea0-112">Verwenden Sie den [.NET Portability Analyzer](../../standard/analyzers/portability-analyzer.md), um Ihre Assemblys zu analysieren und zu ermitteln, ob sie zu .NET Core portiert werden können.</span><span class="sxs-lookup"><span data-stu-id="88ea0-112">Use the [.NET Portability Analyzer](../../standard/analyzers/portability-analyzer.md) to analyze your assemblies and see if they're portable to .NET Core.</span></span>

   <span data-ttu-id="88ea0-113">Das API Portability Analyzer-Tool analysiert Ihre kompilierten Assemblys und generiert einen Bericht.</span><span class="sxs-lookup"><span data-stu-id="88ea0-113">The API Portability Analyzer tool analyzes your compiled assemblies and generates a report.</span></span> <span data-ttu-id="88ea0-114">Dieser Bericht zeigt eine allgemeine Übersicht über die Portierbarkeit und eine Aufschlüsselung nach allen von Ihnen verwendeten APIs, die in .NET Core nicht verfügbar sind.</span><span class="sxs-lookup"><span data-stu-id="88ea0-114">This report shows a high-level portability summary and a breakdown of each API you're using that isn't available on NET Core.</span></span>

3. <span data-ttu-id="88ea0-115">Installieren Sie das [.NET API-Analysetool](../../standard/analyzers/api-analyzer.md) in Ihren Projekten, um APIs zu identifizieren, die eine <xref:System.PlatformNotSupportedException> für einige Plattformen auslösen, und weitere potenzielle Kompatibilitätsprobleme zu ermitteln.</span><span class="sxs-lookup"><span data-stu-id="88ea0-115">Install the [.NET API analyzer](../../standard/analyzers/api-analyzer.md) into your projects to identify APIs throwing <xref:System.PlatformNotSupportedException> on some platforms and some other potential compatibility issues.</span></span>

   <span data-ttu-id="88ea0-116">Dieses Tool ähnelt dem Portability Analyzer, analysiert jedoch nicht, ob Elemente in .NET Core kompiliert werden können, sondern, ob Sie eine API auf eine Weise verwenden, die zur Laufzeit eine <xref:System.PlatformNotSupportedException> auslöst.</span><span class="sxs-lookup"><span data-stu-id="88ea0-116">This tool is similar to the portability analyzer, but instead of analyzing if things can build on .NET Core, it will analyze if you're using an API in a way that will throw the <xref:System.PlatformNotSupportedException> at runtime.</span></span> <span data-ttu-id="88ea0-117">Wenn Sie von .NET Framework 4.7.2 oder höher wechseln, ist dies zwar nicht häufig der Fall, aber eine Überprüfung ist dennoch eine gute Idee.</span><span class="sxs-lookup"><span data-stu-id="88ea0-117">Although this isn't common if you're moving from .NET Framework 4.7.2 or higher, it's good to check.</span></span>

4. <span data-ttu-id="88ea0-118">Konvertieren Sie alle `packages.config`-Abhängigkeiten mit dem [Konvertierungstool in Visual Studio](/nuget/consume-packages/migrate-packages-config-to-package-reference) in das [PackageReference](/nuget/consume-packages/package-references-in-project-files)-Format.</span><span class="sxs-lookup"><span data-stu-id="88ea0-118">Convert all of your `packages.config` dependencies to the [PackageReference](/nuget/consume-packages/package-references-in-project-files) format with the [conversion tool in Visual Studio](/nuget/consume-packages/migrate-packages-config-to-package-reference).</span></span>

   <span data-ttu-id="88ea0-119">Zu diesem Schritt gehört auch die Konvertierung von Abhängigkeiten aus dem `packages.config`-Legacyformat.</span><span class="sxs-lookup"><span data-stu-id="88ea0-119">This step involves converting your dependencies from the legacy `packages.config` format.</span></span> <span data-ttu-id="88ea0-120">`packages.config` funktioniert in .NET Core nicht, daher ist diese Konvertierung notwendig, wenn Paketabhängigkeiten vorhanden sind.</span><span class="sxs-lookup"><span data-stu-id="88ea0-120">`packages.config` doesn't work on .NET Core, so this conversion is required if you have package dependencies.</span></span>

5. <span data-ttu-id="88ea0-121">Erstellen Sie neue Projekte für .NET Core, und kopieren Sie die Quelldateien, oder versuchen Sie, Ihre vorhandene Projektdatei mithilfe eines Tools zu konvertieren.</span><span class="sxs-lookup"><span data-stu-id="88ea0-121">Create new projects for .NET Core and copy over source files, or attempt to convert your existing project file with a tool.</span></span>

   <span data-ttu-id="88ea0-122">.NET Core verwendet ein einfacheres (und anderes) [Projektdateiformat](../tools/csproj.md) als .NET Framework.</span><span class="sxs-lookup"><span data-stu-id="88ea0-122">.NET Core uses a simplified (and different) [project file format](../tools/csproj.md) than .NET Framework.</span></span> <span data-ttu-id="88ea0-123">Sie müssen Ihre Projektdateien in dieses Format konvertieren, um sie weiter nutzen zu können.</span><span class="sxs-lookup"><span data-stu-id="88ea0-123">You'll need to convert your project files into this format to continue.</span></span>

6. <span data-ttu-id="88ea0-124">Portieren Sie Ihren Testcode.</span><span class="sxs-lookup"><span data-stu-id="88ea0-124">Port your test code.</span></span>

   <span data-ttu-id="88ea0-125">Die Portierung auf .NET Core stellt eine erhebliche Änderung Ihrer Codebase dar, deshalb wird dringend empfohlen, Ihre Tests zu portieren. Auf diese Weise können Sie Tests ausführen, wenn Sie Ihren Code portieren.</span><span class="sxs-lookup"><span data-stu-id="88ea0-125">Because porting to .NET Core is such a significant change to your codebase, it's highly recommended to get your tests ported, so that you can run tests as you port your code over.</span></span> <span data-ttu-id="88ea0-126">MSTest, xUnit und NUnit funktionieren in .NET Core.</span><span class="sxs-lookup"><span data-stu-id="88ea0-126">MSTest, xUnit, and NUnit all work on .NET Core.</span></span>

<span data-ttu-id="88ea0-127">Darüber hinaus können Sie versuchen, kleinere Lösungen oder einzelne Projekte mit dem Tool [dotnet try-convert](https://github.com/dotnet/try-convert) in einem einzigen Vorgang in das .NET Core-Projektdateiformat zu portieren.</span><span class="sxs-lookup"><span data-stu-id="88ea0-127">Additionally, you can attempt to port smaller solutions or individual projects to the .NET Core project file format with the [dotnet try-convert](https://github.com/dotnet/try-convert) tool in one operation.</span></span> <span data-ttu-id="88ea0-128">Es kann nicht garantiert werden, dass `dotnet try-convert` für all Ihre Projekte funktioniert, und es kann zu leichten Abweichungen bei Verhalten kommen, das Sie unbedingt benötigen.</span><span class="sxs-lookup"><span data-stu-id="88ea0-128">`dotnet try-convert` is not guaranteed to work for all your projects, and it may cause subtle changes in behavior that you may find that you depended on.</span></span> <span data-ttu-id="88ea0-129">Das Tool sollte als _Ausgangspunkt_ verwendet werden, um grundlegende Elemente zu automatisieren, die sich für eine Automatisierung eignen.</span><span class="sxs-lookup"><span data-stu-id="88ea0-129">It should be used as a _starting point_ that automates the basic things that can be automated.</span></span> <span data-ttu-id="88ea0-130">Es ist keine garantierte Lösung zum Migrieren eines Projekts.</span><span class="sxs-lookup"><span data-stu-id="88ea0-130">It isn't a guaranteed solution to migrating a project.</span></span>
